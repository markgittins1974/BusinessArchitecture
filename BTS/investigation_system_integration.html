🔍 Key Integration Observations

  • Synchronous vs. Asynchronous: Solid arrows = immediate state changes (Salesforce internal); dashed arrows = async APIs (ESYS, Portal, Comms, Email)
  • SLA Gates: C1 (T+0 entry), C4 (T+5 scope lock), C5 (T+10 evidence), C7 (PW-005: 2-day approval), C8 (60-day objection window). All tracked in Investigation_Action_c audit trail.
  • Assessment Memo Non-Response (MSG-010): If no email reply within 5 days (configurable), scheduled job creates escalation task. If still no response at +3 days, escalate to Program Lead.
  • Portal Dual-Flow: Evidence uploads (MSG-007) can happen during AWAITING_CUSTOMER_RESPONSE, and again during AWAITING_OBJECTION if objection lodged and re-investigation triggered (MSG-018).
  • Financial Threshold Routing (MSG-012/MSG-013): Assessment_Liability__c determines approval path: <$100k → Team Lead queue; >$100k → Compliance queue. Threshold check happens at approval entry, not at routing.
  • Closure Loop (MSG-020): IC must reach CLOSED state in Salesforce (60-day objection window expired, no objection received) before PIC_CLOSE API fires. If API fails, manual override available (admin marks PIC closed in ESYS).
  • Email-to-Case (MSG-010): Requires Email-to-Case mailbox configured at investigations-assessments@[domain].gov.au with subject line parsing rule to extract IC reference (regex: [IC-NNNNN]).
  • TRIM Integration (not shown in detail): Investigator enters TRIM_Folder_Path__c on IC (e.g., \\trim-server\investigations\IC-12345). SF validates path on save via TRIM API; if valid, generates hyperlink for investigator access.
  • Day 55 Reminder (not shown): Scheduled job runs daily; when days_until_objection_deadline = 5, sends email reminder to taxpayer via portal notification + creates task for Program Lead to monitor incoming objections.
  • Fallback Paths: If assessment memo never receives response (MSG-010), investigator can manually enter Assessment_Liability__c or escalate to Compliance (skip assessment path). If entry letter fails to dispatch, admin can manually retry or record dispatch via task.
