LOAN Ownership Task Procedure Overview CONTEXT

Purpose

Process Ownership Tasks in LOAN system to validate and complete correct ownership changes in e-Sys based on NOA data. Final stage after Party and Property Tasks resolved.

Why Ownership Tasks Created

After Party (customer) and Property (land ID) are matched, LOAN system attempts to auto-update ownership. When system cannot process due to exceptions or special circumstances, manual Ownership Tasks are created.

Key Definitions

  • VCOO: Verify Change of Ownership (exception detected in automated update)
  • OWN007: Potential duplicate ownership exception
  • OWN022: Owner Unknown record with calendar year boundary issue
  • Terms Sale: Vendor remains owner until purchaser takes possession AND pays 15%
  • Manner of Holding: Joint proprietors vs tenants in common

Time Standard

5 minutes per Ownership Task

Authority: Land Tax Act 2005 s15-s16 | Branch: Land Information Management
Input: Verified Party (customer) and Property (Land ID), ownership update exception
Process: Validate and execute ownership change in e-Sys via DAS verification or manual transfer
Decision: Can LOAN proceed with auto update or requires manual transfer?
Output: Ownership updated in e-Sys, tax liability assigned to correct customer

Q2 - Can the Ownership Change Be Verified? IMPROVED

DAS Bundle Verification (Critical Step)

Before proceeding, verify ownership details in DAS (Dealing system documents) to confirm transfer is correct:

  1. Open separate e-Sys session
  2. Search → Find DAS Bundle (enter Dealing Number)
  3. Go to Land Transfer tab → Verify DOP and TransType
  4. Go to Properties tab → Verify title references match
  5. Go to Transferees tab → Verify buyer/share details
  6. Right-click transferee → View Transferee to see full details

Terms Sale vs Normal Ownership (Decision Matrix) CRITICAL

Terms Sale Definition (LTA2005 s15-s16):
Vendor = deemed owner UNTIL purchaser takes possession AND pays 15% purchase price
Purchaser = deemed owner when in possession (regardless of payment status)

Check DAS: If transfer shows vendor conditions (payment due, possession conditions), may be Terms Sale. Review contract date vs possession date.

Terms Sale Process

  1. Review transfer conditions in DAS Bundle → View Transferee → Contract/Possession dates
  2. If vendor retained conditions (payment due, possession withheld) → Terms Sale
  3. Tick Terms Sale indicator in Purchase Details
  4. Review contract date vs possession date:
  5. - If title updated to new purchaser AND no outstanding LTX → Proceed
  6. - If vendor liabilities outstanding OR possession conditions unfulfilled → Redirect to Business Team Office (BTO) with comment "Possible Terms Sale"

Duplicate Ownership Exception (OWN007) IMPROVED

When OWN007 occurs, investigate:

  • Is same DOP on LOAN task as current DOP in e-Sys? → Duplicate same-day transfer
  • Is same buyer listed on LOAN task as current owner in e-Sys? → Duplicate buyer
  • Is different owner with DOP within 10 days? → Sequence issue (see below)

If duplicate confirmed: Check DAS to understand why two transfers same property same date. Escalate to Senior Staff member if cannot resolve.

Sequence of Ownership (Same-Day or Close Transactions) IMPROVED

Scenario: Two transfers within 10 days with different parties

Procedure Detail: Sequence Resolution Logic for Trust/Trustee, Owner Unknown & Same-Day Transactions▼

Trust → Trustee Sequence (Change of Trustee)

Situation: Current owner = trust, buyer = trustee of same trust

Resolution: Trust is NOT changing ownership; trustee is changing. Update New Possession Date to day BEFORE trust's possession start date. This ensures transfer records in correct sequence.

Trustee → Trust Sequence

Situation: Current owner = trustee, buyer = trust

Resolution: Update New Possession Date to day AFTER trustee's possession start date. Verify trustee name in NOA or View Document button in LOAN (Trust NOA Part C).

Owner Unknown Sequence (OWN022)

Situation: DOP crosses 31 December with Owner Unknown record existing in current year

Resolution Steps:

  1. Review DAS Bundle, TOL signature date, e-Sys dates, CM9 records
  2. If DOP confirmed SAME YEAR as Owner Unknown: Update New Possession Date, click Proceed. Transfer replaces Owner Unknown.
  3. If DOP confirmed FOLLOWING YEAR: Note prior owner CID, click Proceed with following year DOP, then create Land Ownership action item to recalculate prior owner liability for relevant year
  4. If DOP cannot be confirmed → Request Site Valuation or escalate to Senior Staff

Multiple Same-Day Transfers (Joint Ownership Sequence)

Situation: Property transferred to multiple buyers on same day (joint proprietors or tenants in common)

  1. Search COI by dealing number → Review Manner of Holding column
  2. J = joint proprietors (joint and several liability)
  3. S = tenants in common (separate liability per owner)
  4. In LOAN, manually update possession date if DOP differs between e-Sys and source
  5. Verify all buyers linked and manor of holding correct before proceeding

OWNUNKNOWN Source Type

When source NOA type = OWNUNKNOWN: Replaces Owner Unknown record (CID 71948581). Proceed directly with transfer, no DOP conflict expected.

Source: LOAN Ownership Task Procedure; Resolving Exceptions; Change of Trustee; Owner Unknown
Input: VCOO exception, DAS Bundle data, current ownership record
Process: Verify DAS details, identify sequence/exception issue
Decision: Can ownership proceed or requires manual transfer/escalation?
Output: Exception resolved or manual transfer required

Q3 - How Should This Ownership Exception Be Resolved? IMPROVED

Exception Resolution Decision Tree

Exception Type: OWN007 (Duplicate Ownership)

Action:

  1. Search COI by dealing number to understand why duplicate
  2. Check DAS Bundle for two separate transfers OR same transfer recorded twice
  3. If change of trustee scenario (current owner=trust, buyer=trustee) → Adjust possession date (see Q2 sequence)
  4. If change of ownership share scenario (compare Share% to current ownership) → May need Shares tab update instead of new transfer
  5. If genuine duplicate → Escalate to Senior Staff with explanation

Exception Type: OWN022 (Owner Unknown Calendar Boundary)

Action:

  1. Verify DOP using DAS Bundle, TOL signature, and Land Channel
  2. Check if DOP is same year or different year from Owner Unknown record
  3. Adjust possession date if needed to separate transactions (see Q2 sequence)
  4. If DOP prior year → Create Land Ownership action item after proceeding

Exception Type: OWN010 (Terms Sale)

Action:

  1. Review contract date vs possession date in DAS/transfer document
  2. If vendor conditions exist AND vendor liabilities outstanding → Redirect to BTO (comment: "Possible Terms Sale")
  3. If vendor conditions exist BUT title already updated to purchaser → May proceed if no outstanding LTX

Exception Type: OWN015 (Change of Trustee Only)

Action:

  1. Confirm via DAS that beneficiary/trust terms unchanged, only trustee changed
  2. In e-Sys, do NOT update ownership; update Trustees tab of existing trust
  3. Add new trustee with start date = DOP
  4. Cancel Ownership Change in LOAN with reason "Change of Trustee Only"

Exception Type: OWN020 (Manner of Holding Conflict)

Action:

  1. Search COI by dealing number to see current Manner of Holding
  2. Check DAS for intended manner of holding
  3. In LOAN, use Shares tab to verify/update manner of holding if proceeding
  4. If conflict cannot be resolved → Escalate to Senior Staff
If Special Customer flag present, do NOT proceed without Grade 3/4 confirmation (see Q1 CSO SLA). Request confirmation same day if possible.
Procedure Detail: Manual Transfer Prerequisites, Shares Tab Update Logic & Retrospective Period Handling▼

When Manual Transfer Required

LOAN cannot proceed when:

  • Exception unresolvable via date adjustment or sequence fix
  • Multiple overlapping exceptions on same property
  • Special customer automation flag present AND CSO not confirmed
  • Exception code indicates manual processing required (see exception code reference)
  • DAS details conflict with e-Sys data and cannot be reconciled

Manual Transfer Prerequisites in e-Sys

Before manually updating ownership in e-Sys:

  1. DAS Bundle fully reviewed and understood
  2. Current ownership record in e-Sys reviewed
  3. All Party Task details verified (correct customers linked)
  4. All Property Task details verified (correct Land IDs linked)
  5. New Possession Date identified from DAS
  6. New owners' share percentages confirmed from DAS
  7. Manner of holding (joint/tenants) confirmed from DAS
  8. Dealing number or batch reference documented

Shares Tab Update (Share% Change Only)

When duplicate ownership = change of shares only (not new owner):

  1. Compare Share% in LOAN task to current ownership
  2. Go to Shares tab
  3. Highlight customer
  4. Right-click → New Share
  5. Enter new Share% with effective date = DOP
  6. Total must equal 100% (may need to reduce other owners' shares)
  7. Add dealing number to Comments on current ownership record
  8. Then Cancel Ownership Change (not Proceed)

Retrospective Ownership Updates (Prior Assessment Year)

When DOP is in prior assessment year:

  1. After proceeding/manual transfer, recalculate liabilities for prior owner for relevant assessment year
  2. Create 'Land Ownership action item' with:
  3. - Prior owner CID
  4. - Assessment year requiring recalculation
  5. - Non-Issue Indicator ticked
  6. - Comment explaining DOP and ownership change

Q4 - Is Manual Transfer Required in e-Sys? IMPROVED

When to Use Manual Transfer

Use manual transfer when LOAN system cannot automatically update ownership due to exception that cannot be resolved via date/sequence adjustment.

Manual Transfer Process (e-Sys)

  1. Copy Property ID/Land ID from LOAN Linked Property section
  2. Open separate e-Sys session
  3. Search → Find Property (enter Property ID)
  4. Right-click Property on tree → Ownership node
  5. Right-click → Transfer Property
  6. Enter DOP from LOAN task
  7. Enter Dealing Number from LOAN task
  8. Enter Comments documenting reason for manual transfer
  9. Go to New Owners tab
  10. Click Associate Customers → Select buyer(s) from LOAN task
  11. Set Lead Member for joint ownership (first listed owner)
  12. Tick PPR checkbox if principal place of residence
  13. If Shares tab appears, enter Share% per LOAN/DAS
  14. Click Complete

After Manual Transfer Complete

  1. Return to LOAN Ownership Task
  2. Click Proceed with Ownership Change (to mark task complete)
  3. If Prior Assessment Year DOP → Create Land Ownership action item for prior owner recalculation
Reference: Manual Transfer Instructions; e-Sys Property Ownership Update
Procedure Detail: Possession Date vs Contract Date Update Rules & Updating DOP When Incorrect▼

Contract Date vs Possession Date (LTA2005 s15)

Ownership Transfer Triggers:

  • Purchaser = deemed owner when in possession (s15)
  • Vendor = deemed owner until purchaser in possession AND paid 15% (s16)
  • Possession Date: When purchaser takes occupation/control
  • Contract Date: When contract signed (often before possession)

In LOAN: If contract date entered incorrectly (shows future date but possession already occurred), update contract date to day BEFORE possession date.

Correcting Incorrect DOP (Possession Date)

If New Possession Date entered in LOAN, updates DOP when transfer processed.

If DOP incorrect in current e-Sys ownership and need to correct:

  1. Open e-Sys, go to property Ownership node
  2. Right-click current ownership → Edit
  3. Update Possession Date field (if allowed)
  4. Save and exit
  5. This recalculates tax liability for that assessment year

Input: Exception details, decision to proceed manually, all verified details
Process: Execute manual transfer in e-Sys
Decision: Manual transfer complete?
Output: Ownership updated in e-Sys, LOAN task marked complete

Q5 - Can the Ownership Task Be Completed? IMPROVED

Final Completion Checklist

  • ☐ DAS Bundle reviewed and verified
  • ☐ Ownership exception identified and understood
  • ☐ Exception resolution approach determined (proceed via LOAN or manual transfer)
  • ☐ If sequence issue, possession date corrected
  • ☐ If special customer, CSO confirmation obtained
  • ☐ All Party details verified (correct customers linked)
  • ☐ All Property details verified (correct Land IDs linked)
  • ☐ Manual transfer completed (if required)
  • ☐ Land Ownership action item created (if prior year DOP)

Completion Options

Option 1: Proceed with Ownership Change

Click Proceed with Ownership Change button. LOAN system executes ownership update. Ownership Task complete and closed.

Option 2: Cancel Ownership Change

If NOA should not proceed to ownership update:

  1. Click Cancel Ownership Change button
  2. Select Reason from dropdown (e.g., "Change of Trustee Only", "Manual Transfer Completed", "Unable to Process")
  3. Enter Comments explaining reason
  4. Click Cancel
  5. If manual transfer already completed in e-Sys, property ownership already updated (task just marks as cancelled in LOAN)

Post-Completion Documentation

Prior Assessment Year DOP: After ownership updated (LOAN Proceed or manual transfer), create Land Ownership action item to recalculate liabilities for prior assessment year. Include prior owner CID and Non-Issue Indicator.

Reference: LOAN Ownership Task Procedure; Completing or Cancelling Ownership Task; Land Ownership Action Items
Input: Resolved exception, verified ownership details, exception resolution completed
Process: Execute completion via LOAN Proceed or finalize manual transfer
Decision: LOAN proceed or manual already done?
Output: Ownership task complete, ownership updated in e-Sys

Procedure Summary

  • 5-step process (Q1-Q5) for processing LOAN Ownership Tasks
  • Q1: Identify task type (VCOO or Verify Special Customer) via exception code
  • Q2: Verify ownership via DAS Bundle (Transfer, Properties, Transferees tabs)

  • Q5: Complete task via LOAN Proceed or Cancel Ownership Change
  • Critical concepts: Terms Sale (LTA2005 s15-s16), Change of Trustee, Sequence of Ownership, Owner Unknown (calendar year logic)
  • Special handling: Special Customer CSO confirmation (same day SLA), retrospective period recalculation, manual transfer prerequisites
  • Time standard: 5 minutes per Ownership Task (may extend if manual transfer required)
