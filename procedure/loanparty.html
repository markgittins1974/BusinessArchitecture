LOAN Party Task Procedure Overview CONTEXT

Purpose

Process Party Tasks in LOAN system to identify and validate the correct customer (Involved Party/buyer) from Notice of Acquisition (NOA) data. Ensures land tax assessment notices are issued accurately.

Why Party Tasks Created

LOAN system attempts to auto-match buyers to e-Sys customers at Party stage. When auto-matching fails due to exceptions, manual Party Tasks are created for staff resolution.

Key Definitions

  • Involved Party: The purchaser (buyer) from NOA requiring customer matching
  • Source Data: NOA information (name, address, DOB, ACN/ABN)
  • Matching Customers: e-Sys customers identified as potential matches
  • Confidence Level: Quality score (Poor/Low/Medium/High/Very High) for match

Time Standard

5 minutes per Party Task

Authority: Land Tax Act 2005 | Branch: Land Information Management
Input: NOA buyer data (name, address, DOB, ACN/ABN), exception code
Process: Match source data to e-Sys customer or create new customer
Decision: Which customer is correct involved party? Create new?
Output: Customer linked or created, exception resolved

Q1 - What Party Task Type Am I Processing? IMPROVED

Five Party Task Types

Task Type When It Occurs Primary Action
Resolve Party Match No match found OR multiple matches with equal confidence Search further, manually select, or create customer
Verify Party Match Match found but name/address mismatch detected Verify match is correct, update data if needed
Resolve High Volume Party Resolve Party Match + 3+ buyers on NOA Process all buyers; link each individually
Resolve Excluded Party Buyer name contains excluded word (Trustee, Company) Verify entity type, locate or create correctly
Resolve Trust Party Exception Trust NOA; trust or trustee unmatched Create/link trust and trustee (or escalate)

Confidence Level Acceptance Criteria CRITICAL

Very High/High (80-100%): Link directly, verify customer details current
Medium (60-79%): Manual review required, verify match carefully
Low/Poor (0-59%): Reject match, search further or create customer

Q2 - Can I Locate the Correct Buyer/Customer? IMPROVED

Customer Search Sequence (Priority Order)

  1. Step 1: Review Matching Customers in LOAN. If High+ confidence, click Edit Customer to verify current
  2. Step 2: If Low/Medium confidence, search by ACN/ABN (most precise match)
  3. Step 3: Search by DOB + Surname (if ACN/ABN unavailable)
  4. Step 4: Search by Address only (last resort for location matching)
  5. Step 5: Check COI DLS to verify identity before creating new customer
  6. Step 6: If no match found after exhaustive search → Create new customer

Find Customer Function

Click Find Customer button. Source data auto-populates. To search name only, clear Address Format dropdown to blank before searching.

Critical: System Search Behavior

  • Remove non-essential fields: Delete Street Type, Suburb, Municipality to increase match likelihood
  • Address Format dropdown: Set to [blank] to remove address filters and search by name only
  • Refresh button: Prepopulates BOTH name AND address. Use carefully
Do NOT assume top Matching Customers result is current name/address. Always click Edit Customer to verify current details. Multiple name/address records per customer exist.
Procedure Detail: Search Optimization, COI DLS Lookup, ASIC Search, DAS Verification & Create Customer Validation Checklist▼

ACN/ABN Search (Most Precise) - Step 2

  1. Click Find Customer
  2. In Name Details: Click "All" field → enter ACN or ABN
  3. Clear Address Format to [blank]
  4. Click Find

DOB + Name Search - Step 3

  1. Click Find Customer
  2. Name Details: Enter First Name, Surname, DOB
  3. Clear Address Format to [blank]
  4. Click Find

COI DLS Verification (Before Creating Customer)

When: Before creating new customer, verify buyer identity

  • Access MySRO → COI DLS
  • CRITICAL: Uncheck eSys search function
  • Search by Name + DOB
  • If result shows eSys ID match → use that ID
  • If no eSys ID → check if recently created (search LOAN)

ASIC Search (Company/Trust Verification)

When: Buyer is company or trust entity

  • Open separate e-Sys session
  • Tools → View ASIC Files
  • Search by ACN, ABN, or Organisation Name (omit "Pty Ltd", use "%")
  • Verify company is REGISTERED (not deregistered)
  • Check Historic View for former names
  • Decision: If registered & matches source → OK to link/create. If deregistered → Escalate

DAS Bundle Search (Transfer Verification)

When: Verify buyer details match actual transfer; clarify duplicate buyers on same NOA

  • Open separate e-Sys
  • Search → Find DAS Bundle
  • Enter Dealing Number
  • Go to Transferees tab → Right-click → View Transferee
  • Compare to source data
  • Decision: If DAS shows different buyer → reject e-Sys match, search for correct buyer

Create Customer Validation Checklist

Before clicking Create Customer, verify ALL:

  • ☐ Surname in SURNAME field (not middle name)
  • ☐ All available data: First Name, Surname, DOB, ACN/ABN
  • ☐ Current/residential address entered (required)
  • ☐ Address State/Municipality correct
  • ☐ Address Type selected (Residential or Postal)
  • ☐ If PO Box/RMB only → also add valid residential address
  • ☐ COI DLS search completed (no duplicate)
  • ☐ ASIC search completed if company (verify registered)
  • ☐ Customer type correct (Individual/Company/Trust)
  • ☐ No typos in address, proper case
Source: LOAN Party Task Procedure; Find Customer; Create Customer
Input: Buyer source data (name, address, DOB, ACN/ABN)
Process: Search e-Sys database using prioritized search sequence
Decision: Customer found or requires creation?
Output: Customer linked (existing) or created (new)

Q3 - Is Customer Data Complete and Accurate? IMPROVED

Pre-Completion Verification

  • Customer name matches source data (or corrected)
  • DOB present (or applied from source)
  • ACN/ABN present (or applied from source)
  • Future Address added from source data (if provided)
  • Address Type correct (Residential or Postal)
  • Trust entity created and linked (if applicable)
  • All buyers on multi-buyer NOAs linked
  • No duplicate customers exist

Name Change Process

If customer name needs updating:

  1. Ensure customer linked (Link checkbox ticked)
  2. Click Edit Customer button
  3. Names tab → Right-click existing name → Edit, OR
  4. Click Apply Buyer Name (uses source data name)
  5. Click Save and Exit
  6. CRITICAL: After saving, click OFF name then click BACK on it. LOAN refreshes and shows updated name automatically

Address Change Process

If customer address needs updating:

  1. Ensure customer linked (Link checkbox ticked)
  2. Click Edit Customer button
  3. Addresses tab → Right-click address → Edit, OR
  4. Click Add Future Address (if future address in source), OR
  5. Click Add Address (manually add)
  6. To update Delivery Indicator: Right-click address → Edit → tick checkbox
  7. Click Save and Exit
  8. NOTE: Address changes do NOT auto-refresh; use Edit Customer to verify updates
All data must be verified and corrected BEFORE proceeding. Any missing data will cause ownership change to fail downstream.
Procedure Detail: Trust Verification, Bare Trust Handling, Superfund vs Company & Duplicate Resolution SLA▼

Trust Entity Verification

When buyer is a trust: Must locate/create BOTH trust AND trustee

  • Search for trust by name in e-Sys
  • If multiple trusts same name → Match by trustee name (check DAS)
  • If DAS shows CHANGE OF TRUSTEE → May need to update trustee only, not create new ownership

Bare Trust (Cannot be Created in LOAN)

Bare trust with superfund beneficiary: Bare trustee = registered proprietor; superfund = beneficial owner

ESCALATE TO SENIOR STAFF MEMBER - Cannot create in LOAN, must create manually in e-Sys

Superfund vs Company Distinction (Critical!)

  • "ABC Superfund Pty Ltd": COMPANY (has "Pty Ltd" → create as company)
  • "ABC Superfund Trust": TRUST (create as trust, set Sub-Type to "Superannuation Trust (Compliant)")
  • "ABC Superannuation Fund": TRUST (same as above)

Duplicate Customer - Escalation Required

If duplicate customer located:

  1. Confirm customer is truly duplicate (compare name, DOB, address, ACN/ABN)
  2. ESCALATE TO SENIOR STAFF MEMBER IMMEDIATELY
  3. Link only most current/complete record
  4. Document reason in work item notes
  5. DO NOT PROCEED until Senior Staff resolves
Source: LOAN Party Task Procedure; Edit Customer Details; Bare Trusts; Duplicate Customers
Input: Linked customer record, source data
Process: Verify data completeness and accuracy
Decision: Is customer data complete? All buyers linked?
Output: Customer data verified and updated

Q4 - Have All Required Updates Been Applied? IMPROVED

Mandatory Pre-Completion Updates

  • Applied future/buyer address to all buyers
  • Updated surname if spelling differs
  • Applied DOB from source data
  • Applied ACN/ABN from source data
  • Verified trust and trustee linked (if trust buyer)
  • All buyers on NOA linked to customer
  • Address Type set (Residential or Postal)
  • Delivery Indicator set on mailing address
  • No duplicate customers unresolved

Critical System Behaviors

Name Changes Auto-Refresh; Address Changes Don't

After saving name change, click OFF customer name then click BACK = auto-refresh shows updated name. Address changes do NOT auto-refresh; must use Edit Customer to verify.

Edit Customer Button Active Only If Link Ticked

Edit Customer button is ONLY ACTIVE when Link checkbox ticked AND top result for that customer ID highlighted.

Delivery Indicator Must Use Edit, Not Checkbox

Cannot simply tick Delivery Indicator on listed address. Must RIGHT-CLICK address → Edit → tick checkbox → OK.

State Correction Workaround (ACT vs NSW)

If wrong state entered: Remove Suburb and Postcode, then re-enter Postcode and select Suburb from dropdown. System auto-populates correct State.

After all updates complete, click Hide Potentials button to verify correct customer linked before proceeding.
Procedure Detail: Customer Age Indicators, Special Name Issues, Usage Type Selection & Handling Multiple Same Buyers▼

Customer Age Indicators (Without DOB)

If no DOB available, customer ID digit count indicates age:

  • 6 digits: Created before 2000 (older customer)
  • 7 digits: Created 2000-2009
  • 8+ digits: Created 2010+ (newer customer)

Special Name Issues

Surname in Middle Name Field: Check surname field first (required). If missing, check middle name field. Manually move to correct field before creating.

Usage Type Selection (Critical!)

  • Use "Residential": Customer lives at address (home address)
  • Use "Postal": Address differs from home (PO Box, C/O, business mailing address)
  • If PO Box/RMB only: MUST also add valid residential address (e.g., "1 Smith St, Fitzroy")

Multiple Same Buyers on NOA

Situation: Same customer listed twice as buyer on same NOA

Action: Cannot edit in LOAN. Check DAS to confirm customer obtaining share in own right AND as trustee. Manual transfer required in e-Sys. Escalate if unable to resolve.

Input: Linked customer with source data for comparison
Process: Apply mandatory updates
Decision: Are all updates complete?
Output: All updates applied and verified

Q5 - Can the Party Task Be Completed? IMPROVED

Final Completion Checklist

  • All buyers on NOA linked to customers
  • All name updates applied
  • All address updates applied (including future address)
  • All DOB/ACN/ABN applied
  • No duplicate customers unresolved
  • Trust and trustee properly linked (if applicable)
  • Hide Potentials clicked to verify correct matches

Completion Options

Option 1: Proceed with Party Matches (Normal)

Click Proceed with Party Matches button under Actions section. Party Task resolves and routes to Property stage.

Option 2: Cancel Ownership Change (Remove)

If NOA should not proceed:

  1. Click Cancel Ownership Change button
  2. Select Reason from dropdown
  3. Enter Comments explaining reason
  4. Click Cancel
  5. Task removed from system

Task Reservation & Redirection SLA DOCUMENTED

Reserving a Task

When: Task needs to be paused (awaiting info, Senior Staff review)

  1. From My Process Tasks, tick checkbox next to task
  2. Click Reserve button
  3. Enter your name in Work Tray Name
  4. Enter Active Date (reappear date)
  5. Click Reserve

SLA: Do not reserve beyond 5 business days without update/escalation

Redirecting a Task

When: Task requires Senior Staff or specialist review

  1. From My Process Tasks, tick checkbox
  2. Click Redirect button
  3. Enter recipient's name in Work Tray Name
  4. Enter Comments explaining reason
  5. Click Redirect

SLA: Senior Staff ack within 1 day; complete within 3 days

Reference: LOAN Party Task Procedure; Completing or Removing Party Task
