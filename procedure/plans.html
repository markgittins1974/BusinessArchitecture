Q2 - Can plan number work items be created in e-Sys?IMPROVED

⏰ CRITICAL TIMING: CSV must be saved BEFORE 11:30 AM to be picked up by e-Sys batch at 12:00 Noon

📋 SEQUENCE REQUIREMENT: This step MUST be completed before Q3 and Q4. CSV creation enables batch job to create work items overnight, which plan documents attach to in Q3/Q4.

  1. Open Converter File: Open Plans Formula.xlsx from H:\LandTax\Common\Land_tax\Valuations Team\Plans
  2. Paste Email Text: Copy full text from plan delivery email (excluding email headers). Paste into cell A4 of converter.
  3. Extract Plan Numbers: Converter auto-separates plan numbers into column B. Copy all numbers from column B.
  4. Create New Sheet: Open new Excel spreadsheet. Right-click cell A1 → Paste Special → Select "Values" → OK. This converts text to clean numbers.
  5. VALIDATION CHECKLIST (Do NOT skip this):
    • First plan number matches email: _____ (e.g., LP123456)
    • Last plan number matches email: _____ (e.g., PS789012)
    • Total count matches email count: _____ plans
    • NO plan type letters (LP, PS, PC) in column - ONLY numbers
    • NO header rows or extra data in column
  6. Save as CSV: File → Save As → Format: CSV (Comma Delimited) → Filename: PLANS_ddmmyyyy.csv (use email date) → Location: S:\FTP\PLANS → SAVE
  7. Verify Save: Confirm file saved successfully to S:\FTP\PLANS (check file browser)
  8. CLOSE FILE IMMEDIATELY: Close CSV file right after saving. Do NOT leave open or edit further.
  9. Document Time Saved: Note the time CSV was saved. If after 11:30 AM, file will NOT be picked up by noon batch.
  10. Decision: Has CSV been created, validated, saved to correct location, and saved BEFORE 11:30 AM?
CSV VALIDATION DECISION: Do first/last numbers match? Count correct? No letters/headers? If ALL yes → Save. If ANY no → Fix before saving.
CRITICAL: File must be named PLANS_ddmmyyyy.csv with 'PLANS' in capitals. Incorrect filename or location will cause load failure. CSV must contain ONLY numbers - any letters or headers will cause batch job to reject entire file. Time MATTERS - file saved after 11:30 AM will not be processed until next day.
Input data: Plan delivery email, Plans Formula converter template
Processing responsibility: Staff (extraction and validation), e-Sys batch job (overnight work item creation)
Decision: CSV created, validated, and saved before 11:30 AM cut-off (YES/NO)
Output: CSV file in S:\FTP\PLANS; e-Sys batch job creates work items overnight; work items available next morning

Q3 - Are plan images and owners corporation documents available?IMPROVED

📌 SEQUENCE REQUIREMENT: Q3 MUST complete BEFORE Q4. See detail panel for why this sequence is critical.

🌐 TWO SEPARATE SESSIONS REQUIRED: Session 1 (Edge) for plans. Session 2 (Edge In-Private or Chrome) for lot lists in Q4. See "Session Separation" in detail panel for why.

  1. Open Session 1 (Edge): Open Microsoft Edge browser. Navigate to https://www.landata.online/landata.net/landatahome.aspx
  2. Login: Account Number: 322150 → Password: (see Landata credentials section) → Enter 6-digit verification code from valuations email → Tick "Remember for 24 hours" → Login
  3. Navigate to Certificates: Click Property Certificates tab → Click View Certificates link
  4. Set Date Range: In "From Date" field, enter date of plan delivery email. (For weekend batches, check and enter full range, e.g., Friday 01/05 through Monday 03/05)
  5. CERTIFICATE STATUS DECISION POINT (Critical):

    Check each certificate status:

    • IF "Complete": Download this plan's documents immediately
    • IF "Pending": PAUSE this plan. Do NOT download. Document the plan number. Wait 60-90 minutes, then retry this one plan.
    • IF "Failed": DO NOT RETRY. Document plan number, date, status. Escalate immediately to Grade 4 supervisor with full list of ALL failed plans (see Escalation Matrix in Summary view).
  6. Download Complete Plans: For each COMPLETE certificate only:
    • Copy the plan number (e.g., PS123456)
    • Right-click the plan link → Select "Save Link As..."
    • Save to: H:\LandTax\Common\Land_tax\Valuations Team\Plans\Electronic Plans\New Plans
    • Filename format: <PlanType><Number>.pdf (e.g., LP123456.pdf, PS789012.pdf)
  7. Download OC Documents (if present): After plan PDF, check if OC document is available:
    • Single OC: Save as <PlanType><Number>-OC.pdf
    • Multiple OCs: Save as -OC, -OC2, -OC3 (sequentially numbered)
    • Staged/Redevelopment: Append -D1, -S2, etc. (e.g., PS123456-D1.pdf)
  8. OC Matching Note: Mark which OC goes with which plan for matching with lot list in Q4. (You will verify match when downloading lot lists.)
  9. WFH Work-From-Home Guidance: If working from home, download speeds may vary. Allow adequate time for each certificate to download fully. Before moving to next plan, visually verify file appeared in New Plans folder and file size looks reasonable (not 0 KB).
  10. Decision: Have ALL COMPLETE plan certificates been downloaded and saved with correct naming? Have ALL PENDING/FAILED certificates been handled per decision logic above?
Do NOT download Pending certificates - they are incomplete and will cause errors downstream. Do NOT retry Failed certificates - they require investigation and escalation. Failed status means Landata certificate generation failed, not a timing issue.
Input data: Plan numbers from Q2 CSV, Landata account 322150 credentials, plan delivery date
Processing responsibility: Landata (certificate generation), Staff (download and file storage)
Decision: All Complete certificates downloaded; Pending/Failed handled per decision logic (YES/NO)
Output: Plan and OC PDFs saved to New Plans folder with proper naming; ready for Q4 lot list matching
Procedure Detail: Session Separation, Sequencing, and Lot List Matching Logic▼

Why TWO SEPARATE SESSIONS are Required (GAP 3 Addressed)

The Problem: If you use same Landata login session to switch between Property Certificates (plans) and Lot Lists, the session may timeout during the hour-long download process, OR Landata may retain the first account's permissions and reject your lot list search as "wrong account".

The Solution: Use two separate browser sessions simultaneously:

  • Session 1 (Regular Edge): Logged into Account 322150 for Property Certificates tab (plans/OCs)
  • Session 2 (Edge In-Private or Chrome): Logged into Account 319400 for Title Searches tab (lot lists)

How to open In-Private: Right-click Edge icon → Select "New In-Private Window" OR press Ctrl+Shift+P. Opening a new regular window is NOT sufficient.

Why this works: Separate sessions have separate authentication tokens. No cross-contamination of credentials.

Why Q3 Must Complete BEFORE Q4 (GAP 2 Addressed)

Sequence Dependency: After Q3 downloads all plans, you need Q4 to download lot lists and MATCH them with the plans you just saved. If you try to do lot lists first (Q4 before Q3), you'll download lot lists without knowing which plans they belong to, causing reconciliation errors at the end.

Critical matching step: In Q4, you will compare each downloaded lot list with the plan and OC you downloaded in Q3 to ensure they match. This only works if Q3 completes first.

Certificate Status Handling (GAP 4 Addressed)

Complete: Ready to download immediately. Download all Complete certificates.

Pending: Still processing. Retry same plan 60-90 minutes later. If still Pending by end of day, document and escalate next morning.

Failed: Landata certificate generation failed. This is NOT a timing issue. Requires escalation to Grade 4 for investigation. Email list of failed plan numbers to Grade 4 supervisor immediately.

Q4 - Are corresponding lot lists available for all plans?IMPROVED

🌐 SESSION 2 REQUIRED: Use separate browser session (Edge In-Private or Chrome) with Account 319400. Do NOT reuse Session 1 login.

  1. Open Session 2 (In-Private/Chrome): Right-click Edge and select "New In-Private Window" OR open Google Chrome. Navigate to https://www.landata.vic.gov.au/landata.net/ (Chrome) or https://www.landata.online/landata.net/landatahome.aspx (Edge In-Private)
  2. Login Session 2: Account Number: 319400 → Password: (see Landata credentials) → Enter verification code → "Remember for 24 hours" → Login
  3. Navigate to Lot List: Click Title Searches → Click Lot List option
  4. Search Lot List for First Plan: Select plan type (LP, PS, PC, etc.) → Enter plan number (numbers only) → If staged subdivision or redevelopment, INCLUDE stage/redevelopment number → Hit Enter
  5. Save Lot List:
    • When lot list appears, click Print Options button
    • Select "PDF Creator" as printer
    • UNCHECK "After saving open output file"
    • Click Save
    • Save to: H:\LandTax\Common\Land_tax\Valuations Team\Plans\Electronic Plans\New Plans
    • Filename: <PlanType><Number>-LL.pdf (e.g., PS123456-LL.pdf)
  6. MULTI-PAGE LOT LIST DECISION (GAP 8 Addressed):

    After saving lot list, scroll to BOTTOM of Landata page:

    • IF you see "More" link at bottom left: This lot list continues on next page. Click "More" link and save next section as -LL2
    • REPEAT: Click "More", save as -LL3, -LL4, etc. until "More" link NO LONGER APPEARS at bottom
    • WHEN no "More" link appears: You have saved entire lot list. Move to next plan.
  7. Repeat for All Plans: Continue with lot list #2, #3, etc. until lot list downloaded for every plan in original email.
  8. RECONCILIATION CHECK (GAP 5 Addressed):

    Final document count validation:

    Formula:

    ( Original plan count from email ) + ( Number of OC documents ) + ( Number of lot list pages ) = Total files in New Plans folder

    Example: 25 plans + 5 OC docs + 27 lot list pages (some plans had 2-page lot lists) = 57 files total

    If mismatch: Identify which plans are missing lot lists and retry downloads from Landata

    When reconciliation matches: Task complete. Close the converter spreadsheet file.

  9. Archive Email: Move plan delivery email from Valuations inbox to Valuations-Folders → external communication → overnight plans folder.
  10. Decision: Have all plans been matched with corresponding lot lists and reconciliation confirmed?
Each plan MUST have a lot list. Missing lot lists will cause downstream processing failures. Reconciliation mismatch means some files were not downloaded - identify and retry before closing task.
Input data: Plan numbers from Q2, Landata account 319400 credentials, plan documents from Q3
Processing responsibility: Landata (lot list generation), Staff (download, validation, reconciliation)
Decision: All lot lists downloaded, reconciliation complete and matching (YES/NO)
Output: Complete set of plans, OCs, and lot lists in New Plans folder ready for e-Sys upload in Q5

Q5 - What happens to the downloaded files?NEW

✓ You've completed the daily download. Here's what happens next in the process.

  • Files Location: All plan, OC, and lot list files are now in H:\LandTax\Common\Land_tax\Valuations Team\Plans\Electronic Plans\New Plans
  • Next Process Step (Beyond This Procedure): Document upload team will load files from New Plans folder into e-Sys work items created by CSV batch job from Q2.
  • When Upload Occurs: Typically within 1-2 business days of Q4 completion. (Check with Valuations Team for exact timeline.)

This report shows any plan numbers that failed to create work items in e-Sys batch job from yesterday's Q2 CSV creation. If exceptions appear, forward to supervisor.
Post-Processing Handoff: Your work (Q0-Q4) provides the input files and triggers the work item creation. Document upload team (separate team/process) attaches files to work items. You do not need to take further action.
Input data: Completed New Plans folder with all validated documents
Processing responsibility: Staff (completion), Document upload team (next step - outside this procedure), e-Sys (ongoing work item processing)
Decision: Download and reconciliation complete (YES/NO)
Output: Ready for handoff to document upload process; exception report reviewed next business day

Why Procedure Sequencing Matters (Explanation of GAP 2)

Original document said: "The sequence of steps to perform this task... are set for a reason so please do not modify or errors will occur" — but didn't explain the reason.

```

Q0 → Q1 → Q2 → Q3 → Q4 → Q5 Sequencing Dependencies

Q0 (Prerequisites): Must verify access BEFORE starting. If access missing, work halts mid-process.

→ Q1 (Receive): Wait for email. No point creating CSV without plan list.

→ Q2 (CSV Creation): Must complete BEFORE 11:30 AM. Why? Because e-Sys batch job runs at 12:00 Noon and only processes CSVs received before cut-off. This batch creates work items that Q3/Q4 documents will attach to.

→ Q3 (Download Plans): Must complete BEFORE Q4. Why? Because Q4 requires matching lot lists with plans. Cannot match if plans not downloaded first.

→ Q4 (Download Lot Lists): Matches each lot list with corresponding plan from Q3. Also performs final reconciliation count: plans + OCs + lot lists should match total from original email.

→ Q5 (Handoff): Archives email, confirms readiness for next process. Cannot complete until Q4 reconciliation confirms all documents present.

```

File Naming Conventions - Complete Reference

```

CSV File (Q2)

PLANS_ddmmyyyy.csv

Example: PLANS_19042026.csv (for April 19, 2026 batch)

  • dd = two-digit date (01-31)
  • mm = two-digit month (01-12)
  • yyyy = four-digit year
  • PLANS must be in ALL capitals
  • Saved to: S:\FTP\PLANS
  • Format: CSV (Comma Delimited)
  • Content: ONLY plan numbers (no letters, headers, or other data)

Plan Image Files (Q3)

<PT><PlanNumber>.pdf

Examples: LP123456.pdf, PS789012.pdf, PC654321.pdf

  • <PT> = Two-letter plan type (LP, PS, PC, etc.)
  • <PlanNumber> = Numeric plan number only
  • Saved to: H:\LandTax\Common\Land_tax\Valuations Team\Plans\Electronic Plans\New Plans

Owners Corporation Documents (Q3)

<PT><PlanNumber>-OC[#].pdf

Examples: PS123456-OC.pdf, PS123456-OC2.pdf, PS123456-OC3.pdf

  • Single OC: -OC (no number)
  • Multiple OCs: -OC, -OC2, -OC3 (sequential numbering)

Staged/Redevelopment Plans (Q3)

<PT><PlanNumber>-D#.pdf (Redevelopment) ``` -S#.pdf (Subdivision)

Examples: PS123456-D1.pdf, PS789012-S2.pdf, PS654321-S5.pdf

```

Lot List Files (Q4)

<PT><PlanNumber>-LL[#].pdf

Examples: PS123456-LL.pdf, PS123456-LL2.pdf, PS123456-LL3.pdf

  • Single-page lot list: -LL (no number)
  • Multi-page lot lists: -LL, -LL2, -LL3 (sequential for each additional page section)
  • LL must be in ALL capitals
```

Definitions and Acronyms

  • CSV - Comma Separated Values (batch file format for e-Sys)
  • Landata - Victorian Titles Office online information system
  • Plan Number - Unique identifier for registered plan (e.g., PS123456, LP654321)
  • Plan Type (PT) - Two-letter prefix (LP = Lodged Plan, PS = Plan of Subdivision, PC = Plan of Consolidation)
  • OC - Owners Corporation document (unit entitlements for subdivided property)
  • Lot List - List of individual lots/units within a registered plan
  • e-Sys - Electronic system for land tax administration and work items
  • Work Item - Individual task record created in e-Sys by daily batch job
  • Batch Job - Automated process running daily at 12:00 Noon to create work items from CSV
  • LI987 Report - Exception report listing plans that failed to load as work items
  • SRO - State Revenue Office (Victoria)
  • WFH - Work From Home
  • In-Private/Incognito - Browser mode that creates separate authentication session
