Supplementary Valuations - Administration Overview

Purpose

Record, store, register, and report on supplementary valuations (supps) data received from the Valuer General Victoria (VGV).

What Are Supplementary Valuations?

  • Changes to land: Subdivision, buildings erected, etc.
  • Corrections: Incorrect area, AVPCC, other errors
  • Submitted by Valuation Authorities between general revaluations
  • Certified by VGV before delivery to SRO

Two Email Types

  1. 'Confirmation of Receipt': VGV notifies batch received for certification (file for reference)
  2. 'Supplementary Valuations Certified/Processed': VGV delivers certified data (process and store)

Input: Two types of VGV emails with certified supp data
Process: Q1-Q4 workflow: receipt → storage → registration → reporting
Decision: Correct folder? Data captured? Match rates acceptable?
Output: Files accessible for formatting/testing, progress reported

Q1 - Handle Confirmation of Receipt Emails

Process

  1. Navigate to Valuations inbox in Outlook
  2. Locate email titled 'Confirmation of Receipt'
  3. File email into appropriate folder
  4. Email retained for reference (batch in VGV certification queue)
  5. No further action until Certified/Processed email arrives

Purpose

Confirmation emails track which batches are awaiting VGV certification. Retained for audit trail and to follow up if batches go missing.

Key Point: Confirmation of Receipt = FILE FOR REFERENCE ONLY. Actual work begins with Certified/Processed email.
Procedure Detail: VGV Communication Timeline▼

Step 1: Council Submits to VGV

Valuation Authority submits supplementary valuations for certification

Step 2: VGV Sends Confirmation of Receipt

Email notifies SRO batch received. File for reference.

Step 3: VGV Processes and Certifies

VGV reviews and certifies the data

Step 4: VGV Sends Certified/Processed Email

Email with certified data files. Process and store (Q2-Q4).

Input: VGV 'Confirmation of Receipt' email
Process: File email in designated folder for reference
Decision: Email filed? Awaiting Certified/Processed email?
Output: Confirmation stored, batch tracked

Q2 - Store Supplementary Valuations Files

Email Subject Format

[Email Type] – [Municipality] – [VGV Reference]

Example: Supplementary Valuations Certified – Alpine – AL25.SP.01

VGV Reference Format

AL25.SP.01 = AL (Alpine) + 25 (2025 level) + SP (Supplementary) + .01 (sequence)

File Storage - TWO PATHS Based on Timeframe

IF WITHIN TIMEFRAME:

  1. Open VGV 'Supplementary Valuations Certified/Processed' email
  2. Highlight all files (Excel and Word)
  3. Right-click → Save all data
  4. Locate council folder on OneDrive
  5. Create new folder, rename to "yyyy-mm-dd"
  6. Paste data into folder
  7. Move email from 'Registered' to 'Supplementary Valuations Data' folder

IF OUTSIDE TIMEFRAME:

  1. Highlight all files in email
  2. Right-click → Save all data
  3. Locate council folder on OneDrive
  4. Go to 'Unable to apply' subfolder
  5. Create new folder within 'Unable to apply', rename to "yyyy-mm-dd"
  6. Paste data into folder
  7. Move email to 'return date after 31/12/xxxx' folder in Outlook
CRITICAL: File path depends on timeframe. Wrong path = data not accessible. Define "timeframe" with management.
⚠️ GAP: Procedure doesn't define what "timeframe" means. Calendar year? Financial year? Get clarification from management before using this procedure.
Input: VGV 'Certified/Processed' email with file attachments
Process: Extract files, create dated folder on OneDrive (path depends on timeframe)
Decision: Within or outside timeframe? Correct council folder?
Output: Files stored on OneDrive, organized by date

Q3 - Register and Track Supplementary Valuations

Two Register Tables

Table 1: Receipt/Certification Register

Complete when VGV email received:

  • Type (C/M): Country or Metro
  • Council: Municipality name
  • Return Date: From Excel certified tab
  • VGV Reference: From email subject
  • Date Email Received: VGV email arrival date
  • Number Props Certified: From VGV summary
  • Props Not Subject to Cert: From VGV summary
  • Returned to Council: From VGV summary

Table 2: Versions Register

Complete as file progresses through formatting/testing/QA/production:

  • Version Number: Batch version ID
  • VGV References: All refs in batch
  • Date Formatted/Tested: Completion date
  • Allocated for QA: Officer assigned
  • Date Quality Checked: QA completion
  • Date Signed Off: Approval date
  • Date Loaded to Production: Load completion
  • Test Match Rate %: (1 - test mismatches/properties)
  • Actual Match Rate %: (1 - actual mismatches/properties) ✓ Must be ≥85%

Match Rate Requirement

Test Match Rate: Keep > 85% (smaller supp files can vary)

Actual Match Rate: Must be ≥ 85% (CRITICAL - e-Sys typically higher than test)

Register Location: OneDrive > State Revenue Office > LGAT - Supps Formatting > [Current Year]
Input: VGV certified data, formatting/testing/QA completion dates
Process: Update Receipt table, track through Versions table as file progresses
Decision: All data captured? Match rates acceptable?
Output: Progress tracked, files registered

Q4 - Report Supplementary Valuations Progress

End of Month Report Content

  1. Total Files Received and Loaded: Count entries in modified tab (received), count with load date (loaded)
  2. Receipt from Each Council: Check pivot tables tab, escalate to VGV if council missing by January
  3. Version 1 Batches Outstanding: Calculate in Batch 1 table in Supps Schedule tab
  4. Current Test Match Rate: Average Test Match Rate in cell N2 (modified tab)
  5. Actual Match Rate: Actual Match Rate in cell R2 (modified tab) - MUST be ≥85%

Report Recipients

  • LGA Team Leader
  • LRB Branch Manager
  • Timing: End of each calendar month

Data Sources in Register

  • 'Modified' Tab: Test and actual match rates (cells N2, R2)
  • 'Supps Schedule' Tab: Batch 1 table (v1 batches outstanding)
  • 'Pivot Tables' Tab: Council receipt tracking
  • 'Graphs' Tab: Supplementary data loaded graph
Automated Data Flow: Supp files loaded daily automatically entered into register, data flows to graphs tab automatically.
Input: Supps register data (all tabs)
Process: Extract metrics, compile summary, identify missing councils
Decision: Data current? Match rates acceptable? All councils covered?
Output: End of month report to leadership

Critical Gaps Identified - 7 Issues

🔴 GAP 1 (HIGH): "Timeframe" UNDEFINED - Procedure says store "within timeframe" vs "outside timeframe" but never defines it. Calendar year? Financial year? Assessment cycle? Staff may route files to wrong folder.
🔴 GAP 2 (HIGH): AUTOMATION FAILURE HANDLING MISSING - Procedure mentions "automation processed with error" but no fallback procedure documented. No error resolution. No escalation.
🟡 GAP 3 (MEDIUM): MATCH RATE RELATIONSHIP UNEXPLAINED - Actual match rate usually higher than test match rate, but why? E-Sys "more up to date" than test DB but relationship not clear. Staff confused during QA.
🟡 GAP 4 (MEDIUM): COUNCIL FOLLOW-UP VAGUE - "By January if no files from council, follow up with VGV" but who? How? When? No documentation template or escalation process.
🟡 GAP 5 (MEDIUM): WEBMAIL FALLBACK NOT EXPLAINED - Check Webmail if files missing from desktop Outlook, but when? Why? No sync process documented.
🟠 GAP 6 (LOW): NRNL NOT DEFINED - "Total Props Loaded (NRNLs removed)" but acronym not explained. Non-Rateable Non-Leviable? Why removed? New staff confused.
🟠 GAP 7 (LOW): ONEDRIVE AUTOMATION INCOMPLETE - How does automation work? What triggers it? Error conditions? Recovery steps? Not documented.

Procedure Summary

  • Q1: File Confirmation of Receipt emails for reference (batch in VGV queue)
  • Q2: Extract Certified/Processed data, store to OneDrive with date folder (path depends on timeframe)
  • Q3: Register in Receipt table, track through Versions table, maintain match rates >85% test, ≥85% actual
  • Q4: Report monthly: files loaded, council coverage, match rates to leadership
  • Critical Threshold: Actual match rate MUST be ≥85%
  • Register Location: OneDrive > State Revenue Office > LGAT - Supps Formatting
