Returned Unclaimed Amendments Procedure Overview

Purpose

Identify and process returned unclaimed (RTU) Land Tax assessments and correspondence by locating current addresses, updating e-Sys records, and reissuing assessments or correspondence as needed.

Time Standard

15 minutes per RTU item

Key Steps

  • Cross-reference resources to locate current address and identify PPR exemption
  • Update information in e-Sys database
  • Check correctness of land tax liabilities and assessments
  • Cancel, amend, or recalculate liabilities as needed
  • Issue correspondence if necessary

RTU Action Items Created For

  • Land Tax Assessment Notices
  • Land Tax Instalment Notices
  • Correspondence from SRO staff to customers
  • Debt Management Services (DMS) letters
  • Auto-generated letters from the Commissioner
Authority: Land Tax Act 2005 | Branch: Land Information Management | Source: April 2024
Input: RTU action item (scanned mail returned unclaimed) from Business Support Team
Process: Research customer/property, locate current address, update e-Sys, issue/cancel assessments
Decision: Which scenario applies (Individual/Joint, Company/Trust, PPR, DMS, Unable to Locate, Cancel)?
Output: RTU action item complete with updated records and reissued correspondence

Q1 - Is Customer Under Investigation?

Investigation Check

CRITICAL: Before proceeding, check if customer is under investigation by checking e-Sys for non-issue "Under Investigation".

If Under Investigation: DO NOT process RTU amendment. Escalate to Investigation Branch.

Non-Issue Indicators

  • Red exclamation mark (!) next to customer number
  • Non-issue type: "Under Investigation"
  • Investigation number (prefixed IC)
Do NOT update records or reissue assessments for customers under investigation. Escalate immediately.
Procedure Detail: Escalation Process for Investigation Customers▼

Investigation Escalation

  1. Identify investigation number (IC prefix)
  2. Document investigation number in RTU action item notes
  3. Reassign RTU action item to Investigations Branch
  4. Close action item with note: "Under investigation - refer to Investigations"
Input: RTU action item with customer details
Process: Check e-Sys non-issue indicators for investigation status
Decision: Is customer under investigation?
Output: If YES - escalate to Investigations; If NO - proceed to Q2

Q2 - What Type of RTU Scenario Applies?

Seven RTU Scenarios

Scenario Customer Type Primary Action
Individual/Joint Ownership Individual or joint customer Locate current address, update e-Sys, reissue
Company/Trust Company or trust registered as owner Verify ABN/ACN, locate address, update e-Sys
PPR Exemption Customer claiming PPR exemption Verify PPR claims, check prior liabilities
Debt Management (DMS) Customer in debt recovery Verify receipt, update address, coordinate with DMS
Unable to Locate Address No alternative address found Continue research via multiple sources
Cancel Assessment Assessment never received by customer Determine if cancellation appropriate, cancel if needed
Managing Agent Property has active managing agent Locate agent, redirect assessment to agent

How to Determine Scenario

  1. Review RTU action item details (customer type, property, assessment type)
  2. Check e-Sys for special flags (PPR, Debt Recovery, Managing Agent)
  3. Review correspondence type (assessment, instalment, DMS letter)
  4. Select appropriate scenario from decision tables below (Q3)

Procedure Detail: Research Tools Explained & Search Sequence▼

Primary Research Tools

  • e-Sys: Check customer records, property ownership, liability status
  • My Land Tax (MLT): Check if customer registered for online portal
  • Muni Returns: Municipal valuation data for address verification
  • Live DLS: Current Driver License address (requires privacy check)
  • Landata: Land title information from Titles Office
  • COI (Compliance Operational Intelligence): Customer/property/title data warehouse
  • ASIC: Company/trust registration verification
  • RTBA: Real estate bond information and agent registration

Recommended Search Sequence

  1. e-Sys for current customer/property records
  2. Muni Returns for council valuation address
  3. Landata for title reference verification
  4. COI for consolidated customer data
  5. Live DLS if customer consent confirmed
  6. If company/trust: ASIC verification
Input: RTU scenario identified in Q2
Process: Use decision table to research customer/address/property details
Decision: What address/data research is needed? Which tools to use?
Output: Research complete, address located or alternative approach determined

Q4 - What e-Sys Records Should Be Updated?

Update Actions by Scenario Outcome

Based on decision table outcome from Q3, perform following updates in e-Sys:

Standard Updates (All Scenarios)

  • Update customer mailing address to current/alternative address
  • Update any corrections to property address or references
  • Remove or add No Postal Address NII if appropriate
  • Document RTU action in customer notes with date and resolution

Scenario-Specific Updates

  • PPR: Apply/amend PPR exemption, recalculate liability
  • DMS: Update address, notify Debt Management Services, may resolve recovery
  • Managing Agent: Add managing agent to property records, redirect correspondence
  • Cancel Assessment: Cancel assessment, add NII "Assessment Not Received", consider penalty waiver
  • Penalty Created: DO NOT cancel yourself; escalate to Senior Officer

Reissuing Assessments/Correspondence

  1. After updating e-Sys, determine if assessment/correspondence should be reissued
  2. If assessment prior to current year: Create "ASSESSMENT – LATE ISSUE OF" letter
  3. Generate new assessment or copy correspondence letter
  4. Issue to updated address (postal or new mailing address)
  5. Document reissuance in action item notes
If penalty item created by amendments, DO NOT cancel. Escalate to Senior Officer immediately for penalty tax review.
Procedure Detail: Penalty Handling, NII Management & Late Assessment Letters▼

Penalty Tax Items

If your updates create a penalty tax item (shown in Penalties node), you cannot cancel it yourself.

  1. Document item in action item notes
  2. Escalate to Senior Officer for penalty tax review/cancellation
  3. Do NOT proceed with assessment reissuance until penalty resolved

Non-Issue Indicator (NII) Management

  • No Postal Address: Add NII if address cannot be located after exhaustive search
  • Assessment Not Received: Add NII if assessment cancelled due to non-receipt
  • Cease/Remove: Remove NII when alternative address located and updated

Late Assessment Letter

If assessment is prior to current year and being reissued:

  1. Create letter: "ASSESSMENT – LATE ISSUE OF [YEAR]"
  2. Explain why assessment is being reissued (RTU resolved)
  3. Include current payment arrangements and due dates
  4. Send with reissued assessment
Input: Research outcome from Q3, decision table result
Process: Update e-Sys records (address, PPR, NII, etc.)
Decision: Which records to update? Generate late letter?
Output: e-Sys updated, assessments/correspondence generated for reissuance

Q5 - How Should the RTU Action Item Be Completed?

Action Item Completion Steps

  1. Ensure all e-Sys updates complete (address, property refs, PPR, etc.)
  2. Generate new assessment/correspondence letter (if required)
  3. Verify new mailing address in e-Sys
  4. Document outcome in RTU action item notes
  5. Mark action item as complete
  6. File reissued correspondence in scan system

Completion Documentation

In RTU action item notes, document:

  • Research conducted and tools used (e.g., "Verified via Muni Returns and COI")
  • Alternative address located: [Address]
  • e-Sys updates applied (address, PPR, NII removal, etc.)
  • If assessment cancelled: reason why (e.g., "Never received by customer")
  • If reissued: assessment type and reissue date
  • If unable to locate: note "Exhaustive search complete, NII applied"
  • Any escalations: Senior Officer review for penalties, Investigations, etc.

When Unable to Complete

If unable to locate address after decision table research:

  • Place "No Postal Address" NII on customer account
  • Document search efforts in notes
  • Consult Senior Officer before escalating further
  • Escalate RTU action item if requiring further investigation

Decision Tables Summary

Seven decision tables guide research for different RTU scenarios. Each table shows: Consideration → Research → Tools → Outcome

  • Individual/Joint Ownership: Check MLT registration, locate current address, update e-Sys, reissue
  • Company/Trust: Verify ABN/ACN, confirm registration with ASIC, locate address, update records
  • PPR Exemption: Verify PPR claims, check address history, apply/amend exemption, recalculate liability
  • Debt Management (DMS): Verify receipt, locate alternative address, update DMS coordination, potentially resolve recovery
  • Unable to Locate Address: Exhaustive search via multiple sources, place NII if address cannot be found, consult Senior Officer
  • Cancel Assessment: Determine if never received, confirm via DLS/DMS, cancel if appropriate, waive penalties if needed
  • Managing Agent: Confirm active bond, locate agent contact, contact agent, redirect assessment/update e-Sys

Procedure Summary

  • Purpose: Process returned unclaimed mail by locating current addresses and reissuing assessments/correspondence
  • Q1: Check if customer under investigation (escalate if YES)
  • Q2: Identify which of 7 RTU scenarios applies
  • Q3: Use decision table to research customer/address/property details
  • Q4: Update e-Sys records (address, PPR, NII, etc.) and generate reissued correspondence
  • Q5: Complete RTU action item with documented outcome
  • Time standard: 15 minutes per RTU item
  • Critical escalations: Investigation customers, penalties, unable to locate address
