EPIC: Work Allocation & Capacity Management
Work Demand Management
1. Structured Backlog Intake

As a FIMB Analyst, I want to load validated PICs in a structured way so that demand is consistently captured and ready for prioritisation.

2. Prioritised Backlog Visibility

As a Team Leader, I want PICs organised into a prioritised backlog so that I can allocate work efficiently.

Acceptance Criteria
  • Validated PICs loaded into a single backlog
  • Each PIC has ID, validated status, ready for allocation
  • Supports sorting (risk, age, revenue type)
  • Visible across roles
  • No spreadsheets required
  • All backlog changes recorded
3. Automated Demand Evaluation

As the system, I want to evaluate PICs so that simple cases auto-progress and complex cases go to review.

Acceptance Criteria
  • All PICs evaluated using configurable rules
  • Outcome recorded (Auto / Manual)
  • Rules are auditable
  • Auto cases progress immediately
  • Manual cases routed with context
  • No bypass or external tools
4. Auto vs Manual Segmentation

As a Team Leader, I want visibility of auto vs manual PICs so that effort is focused appropriately.

Acceptance Criteria
  • PIC labelled Auto or Manual
  • Manual PICs appear in queue
  • Auto PICs require no intervention
  • Team-level visibility of split
5. Configurable Evaluation Control

As a Manager, I want to enable or disable rules so that processing can be controlled.

Acceptance Criteria
  • Evaluation can be enabled or disabled
  • Setting is configurable and auditable
  • Disabled routes all PICs to manual
  • Existing PICs not impacted
Work Allocation
6. Queue-Based Routing

System routes PICs to queues so work reaches correct teams.

Acceptance Criteria
  • All manual PICs routed
  • Rules based (type, priority)
  • Each PIC assigned to one queue
  • No unassigned PICs
7. Queue Visibility

Team Leaders can view queues to balance workload.

Acceptance Criteria
  • Queue shows volume and status
  • Real-time visibility
  • Imbalances identifiable
8. Configurable Queue Rules

Managers can configure routing rules.

Acceptance Criteria
  • Rules editable without code
  • Changes auditable
  • Apply to new PICs
9. Officer Assignment

System assigns PICs to officers.

Acceptance Criteria
  • PIC assigned to officer
  • Based on workload, skill
  • Clear ownership recorded
  • No manual step when rules active
10. Balanced Distribution

Work distributed evenly.

Acceptance Criteria
  • Workload visible per officer
  • Imbalances identifiable
  • Ownership traceable
11. Configurable Assignment Rules

Business controls assignment logic.

Acceptance Criteria
  • Rules configurable without code
  • Based on workload, role
  • Auditable changes
12. Manual Reassignment

Team Leaders can manually reassign work.

Work Notification
13. Assignment Notification

Officer notified when work assigned.
