✗ Out of Scope (23 Capabilities)

What it means: Features that POC does NOT include. These are upstream (intelligence, intake automation), downstream (objections, post-closure), external (system integration, APIs), or infrastructure concerns (data pipelines, governance).

Why: POC validates ONLY case management core. Everything else is deferred to Phase 2+ or owned by other teams.

Examples: Intelligence & Referral • Risk Indicators • Data Integration • Automated & Bulk Initiation • External Search • Cross-Revenue Coordination • Knowledge Management • Technology & Systems Integration • Policy Governance • and 14 more...

📋 Technical Enablers (In Scope)

Required to deliver POC: Data mapping & CSV load (accounts, seed PICs) • Case record types (PIC/IC) • Assignment rules & escalation (SLA enforcement) • List views & record views • Role hierarchy & permissions • Salesforce REST APIs (read-only eSys context) • Manual eSys sync (daily cache) • Data Loader validation

NOT Included (Phase 2+): MuleSoft middleware • Real-time webhooks • Automated ETL pipelines • System integrations • Bulk API 2.0 • SSO integration

🎯 Success Criteria

POC Succeeds When:

  • ✓ Adoption >80% (investigators daily active users in Salesforce)
  • ✓ Approval cycle improves 5d → 2d (avg, 80% within SLA)
  • ✓ Case closure cycle improves 15%+ (toward 60d target)
  • ✓ Team leads see real-time visibility (dashboards adopted)
  • ✓ All 7 capabilities operationally stable (no critical bugs)

Green Light: ≥4/5 criteria met → Approve Phase 2 automation & integration.

🎯 Key Principle

POC validates the case management core is sound. It does NOT attempt to solve: upstream intake automation, external data integration, advanced workflows & AI, infrastructure concerns, or governance/policy frameworks.

Success = Investigators adopt Salesforce for case documentation, TLs approve via workflow, executives see real-time visibility. Everything else is Phase 2+.
