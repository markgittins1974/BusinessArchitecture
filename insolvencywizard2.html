Insolvency Decision Wizard

Land Tax Insolvency Procedure - Step-by-step guided assessment

Q1 - IP Correspondence
What revenue line does this IP request relate to?
Check the IP correspondence. If documentation is older than 6 months, verify insolvency status via ASIC/VEDA before proceeding.
Q2 - Customer Registration
Is the customer registered in e-Sys?
Search using identifiers from IP correspondence. If not found: check IP correspondence for correct entity, ABN Lookup, ASIC search, then Equifax/VEDA search.
Q3 - Customer Status
Is the customer still insolvent?
Check ASIC / VEDA. If the customer is no longer bankrupt, in administration, etc. — or if time has passed and status has changed — they may no longer be insolvent.
Q3 - Land Ownership
Does the customer currently own land?
Check the Land Owner node (MC322). For trustees: check if the trust owns properties — load associated Trustee/Trust accounts into customer tree. Use ASIC/COI/LTO to verify.
PRIVACY: Only release information directly related to the IP request. If IP-provided details conflict with e-Sys, send only relevant correspondence.
Q3 - Outstanding Debt
Does the customer have outstanding land tax debt?
Check liabilities under Land Taxpayer node. Scroll to Net Liability column in the Assessment Year Liability Summary.
Q4 - Appointment Type
What type of insolvency appointment applies?
Check the IP correspondence for appointment type. Verify against (IP) Managed Customer Node in e-Sys. If IP not in system, check IP Listing Spreadsheet (One Note, Insolvency TAB).
Q5 - Debt Timing
Is the debt pre-appointment, post-appointment, or both?
Pre-appointment: liabilities payable as at midnight 31 December before appointment date (full year, not apportioned). Post-appointment: assessments on or after 31 December after appointment date. Check suspense and allocate to outstanding liabilities.
Ensure all assessments for pre and post-appointment debt have been issued before sending correspondence.
