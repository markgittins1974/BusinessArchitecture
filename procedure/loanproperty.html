LOAN Property Task Procedure Overview CONTEXT

Purpose

Process Property Tasks in LOAN system to identify and validate the correct property (Involved Property) from Notice of Acquisition (NOA). Match property references to e-Sys Land IDs.

Why Property Tasks Created

LOAN system attempts to auto-match property references to e-Sys Land IDs at Property stage. When auto-matching fails due to exceptions, manual Property Tasks are created.

Key Definitions

  • Involved Property: The property from NOA requiring e-Sys Land ID matching
  • Title References: Volume/Folio, Lot/Plan, Crown References
  • SPI: Spatial Property Identifier (matches municipality/title)
  • Part Indicator: When same title reference on multiple properties

Time Standards

Resolve Property Conflict: 3 mins | Resolve Property Match: 4 mins | All other: 5 mins

Authority: Land Tax Act 2005 | Branch: Land Information Management
Input: NOA property data (address, title references, SPI, municipality)
Process: Match property references to e-Sys Land ID
Decision: Which Land ID is correct property? Create/update?
Output: Property linked or created, references verified

Q1 - What Property Task Type Am I Processing? IMPROVED

Six Property Task Types

Task Type When It Occurs Primary Action
Resolve Property Match No match OR multiple matches equal confidence Find property manually or create
Resolve Mult Property Match Resolve Property Match + 2 source properties Locate both properties, link each
Resolve Property Conflict Match found but title/address mismatch Verify match, update references/address
Resolve Mult Property Conflict Resolve Property Conflict + 2 properties Verify both, update as needed
Resolve High Volume Property High number of source properties on NOA Process all properties individually
Resolve Property in Parcel Property is part of parcel with other properties Verify all parcel properties transferring

Reference/Address Score Thresholds CRITICAL

Very High/High (0.8-1.0): Link directly, verify references
Medium (0.6-0.79): Manual verification required
Low/Poor (0-0.59): Reject, search further, use Muni Returns for verification
Procedure Detail: Task Type Decision Matrix & Reference Score Methodology▼

Reference Score Components

Range: 0 to 1

  • Volume/Folio exact match = high score component
  • Lot/Plan exact match = high score component
  • Crown References match = high score component
  • Historical or incorrect references = reduce score

Address Score Components

Range: 0 to 1

  • Street number/name exact match = high score
  • Suburb/postcode match = moderate score
  • Minor variations (e.g., "Mount" vs "Mt") = reduce score
  • Address history changes = consider start date on property

Q2 - Can I Locate the Correct Property? IMPROVED

Property Search Methods (Priority)

  1. Step 1: Review Matched Properties in LOAN. If High+ confidence, verify via Edit Property Address/References
  2. Step 2: If no match, use Find Property (by Address/Municipal or Property References)
  3. Step 3: Use Find Seller's Properties (locate via owner/customer)
  4. Step 4: Search Muni Returns (Municipal Revaluation Files) for title verification
  5. Step 5: Search COI for title reference verification and status
  6. Step 6: If still no match → Request Site Valuation

Find Property Search Screen

Address/Municipal Section: Prepopulated with source data. Remove Street Type/Suburb/Municipality to improve search likelihood.

Property References Section: NOT prepopulated. Enter Volume/Folio or Lot/Plan manually to search by reference.

Muni Returns Search (Municipal Revaluation Files) SLA DOCUMENTED

Access: Tools → View Municipal Revaluation Files

Search by: SPI, Owner, Street name (select most recent year for current details)

Output: Returns Property ID in e-Sys that matches council valuation data

Site Valuation Request (For Complex Cases) SLA ADDED

When: Cannot locate property after exhausting all searches; need pro-rata subdivision confirmation; need title reference clarification

Action: Click "Request Site Valuation" button. Complete Dealing Number, References, and detailed Comments explaining what action is required.

SLA: Typically 1-2 weeks for response. Check Work Item Notes. Reserve your Property Task while awaiting.

Do NOT assume property with "no owner" has been subdivided. Verify in COI or via Site Valuation Request. Title references with "Lodged Plan" type may not search correctly.
Procedure Detail: COI Search, Find Seller's Properties, Parcel Property Handling & Special Address Issues▼

COI Property Search

Access: MySRO → COI Main Menu → Find Property

  • Search by Dealing Number (to get all property references)
  • Search by Volume/Folio (exact match for title)
  • Search by Lot/Plan (may return all plan types, including Lodged Plans)
  • Search by SPI (confirms property details)

Output: Shows registered proprietor, active/historical references, recent dealings

Find Seller's Properties Function

When: Know customer/seller, need to find their properties

  1. Click Find Seller's Properties
  2. Click Refresh to prepopulate seller data from NOA
  3. Or search by Customer ID, Name, or Address
  4. Select customer → Click OK
  5. Property Ownerships display → Select correct property
  6. Returns to LOAN with property linked

Properties in Parcel Handling

When: Property is contained within parcel, may need to verify all parcel properties transferring

  1. If linking property in parcel, warning popup appears
  2. Load property in separate e-Sys session
  3. Select Parcels node → view Parcel Summary
  4. Double-click parcel to view Parcel Detail
  5. Double-click each parcel property to load in tree
  6. Review title references for each, verify via COI if active
  7. Return to LOAN and link ALL parcel properties (using LIDs)
  8. Verify via title references that all parcel properties are transferring
  9. If not all transferring: Consult with Business Team Office (BTO)

Special Address Issues

  • Mt vs Mount: LOAN does not recognize as equivalent. May cause "different street name" error. Verify address manually.
  • Recent Subdivision (start date <18 months): Address may not match council data. OK to proceed without confirming address via Muni Returns.
  • No Owner Shown: May indicate property subdivided. Check COI for children properties. Manually transfer children if needed.
  • Easement vs Property: Check Land Channel for sale price/size. If very small with small sale price → may be easement. Cancel ownership change with reason "Properties with little Significance".

Input: Property source data (address, title references, SPI)
Process: Search e-Sys using multiple methods (Find Property, Muni Returns, COI)
Decision: Property found or requires Site Valuation Request?
Output: Property linked or Site Valuation Request submitted

Q3 - Are Property References Correct and Complete? IMPROVED

Reference Verification Checklist

When property located, review title references in Matched Properties section:

  • ☐ Volume/Folio matches source data or is alternate valid reference
  • ☐ Lot/Plan matches source data
  • ☐ Crown References match (if applicable)
  • ☐ References are ACTIVE (not historical/inactive)
  • ☐ If multiple references, understand "part indicator" usage
  • ☐ Verify via Muni Returns if confusion about split references

Title Reference Special Cases CRITICAL

Case 1: Same Title Reference on Multiple Properties

Confirm via Muni Returns SPI that references belong to each property. If yes, tick "Part Indicator" checkbox on ALL properties containing reference.

Case 2: Lodged Plan Type (L prefix)

LOAN search results will show ALL plan types matching lot/plan (not just Lodged Plans). May need to filter manually or match using Address instead. Cannot add Lodged Plan reference to property unless one already exists.

Case 3: Reference Currently on Another Property

If trying to add reference and error says "currently on another property" → Verify via Muni Returns/COI that reference belongs to both properties. If yes, manually delete from other property first before adding to your property in LOAN.

Case 4: Additional/Extra References

If property has references not in source data → Check COI/Muni Returns: Inactive? Remove. Active and accessory? Add. Mistake? Remove. Need pro-rata? Request Site Valuation.

When editing property references in LOAN, changes apply immediately. But if must delete reference from one property THEN add to another, must Save and Exit after delete before adding to second property.
Procedure Detail: Possession Date Updates & Edit Property Operations▼

Updating Possession Date

When: Linked property start/end date doesn't match possession date from source

  1. Go to Purchase Details section (top middle)
  2. Existing Possession Date shows source data date
  3. Enter New Possession Date (to correct the ownership change)
  4. Click Apply

Edit Property Address

When: Property address needs updating to source data address

  1. Click Edit Property Address button (only active if Link ticked)
  2. Current address from e-Sys displays
  3. Click Copy Source Property Address (to update to NOA address), OR
  4. Manually edit address fields
  5. Click OK
  6. Confirm address on Muni Returns

Edit Property References

When: Title references need adding/removing

  1. Click Edit Property References button
  2. Left side: Source/Available references. Right side: Matched Property references
  3. To ADD: Highlight reference on left → Click arrow pointing right
  4. To REMOVE: Highlight reference on right → Click arrow pointing left
  5. Tick "Part Indicator" if same reference on multiple properties
  6. Click OK
Source: LOAN Property Task Procedure; Edit Property Details
Input: Linked property record, source references for comparison
Process: Verify reference accuracy and completeness
Decision: Are references correct? Need updates?
Output: References verified and updated

Q4 - Have All Property Updates Been Applied? IMPROVED

Mandatory Property Updates

  • ☐ Property address matches source data (or verified via Muni Returns)
  • ☐ All title references added (Volume/Folio, Lot/Plan, Crown)
  • ☐ Inactive references removed (if applicable)
  • ☐ Part indicator ticked if reference on multiple properties
  • ☐ Possession date corrected (if different from property start date)
  • ☐ All properties in parcel linked (if property in parcel)
  • ☐ No duplicate property records unresolved

Before Proceeding

Verify all visible property references in Property References section match source data. If references differ slightly (plan type, format), verify via Muni Returns or COI that property is correct.

Click Show Potentials button to review lower-confidence matches and ensure you selected correct property.

Reference: LOAN Property Task Procedure; Edit Property Details
Procedure Detail: Subdivision/Consolidation Handling, SPI Matching & Property End Date Interpretation▼

Subdivision/Consolidation Scenarios

Property Recently Subdivided:

  • Old parent property will show "no owner" in e-Sys
  • New child properties created
  • Check COI for children properties with same parent references
  • Manually transfer to children if required by NOA

Property Being Consolidated:

  • Multiple references on single property
  • Verify SPI matches across all references via Muni Returns
  • Add all references to single e-Sys property

SPI Verification (Key Step)

SPI (Spatial Property Identifier) from Muni Returns shows: SPI = [Number]-[Number]-[Number]

  • Check SPI from source data against Muni Returns SPI
  • If same SPI → References belong together on single property
  • If different SPI → References belong to different properties

Property End Date Interpretation

When Property End Date Shown:

  • Property has been deleted/voided from e-Sys
  • Cannot transfer deleted property
  • May indicate recent subdivision or consolidation
  • Check children properties if subdivision suspected
  • Escalate if cannot resolve

Input: Linked property with references for verification
Process: Apply mandatory updates
Decision: Are all updates complete and correct?
Output: All updates applied and verified

Q5 - Can the Property Task Be Completed? IMPROVED

Final Completion Checklist

  • ☐ All source properties linked to e-Sys Land IDs
  • ☐ All address updates completed and verified
  • ☐ All title references correct and complete
  • ☐ Part indicator applied where needed
  • ☐ Parcel property handling verified (if applicable)
  • ☐ Possession date updated (if needed)
  • ☐ No duplicate property records
  • ☐ Show Potentials reviewed to verify correct property selected

Completion Options

Option 1: Proceed with Property Matches

Click Proceed with Property Matches button. Property Task resolved and routes to Ownership stage.

Option 2: Cancel Ownership Change

If NOA should not proceed:

  1. Click Cancel Ownership Change button
  2. Select Reason from dropdown
  3. Enter Comments
  4. Click Cancel

If Cancelled: May need to manually transfer property. Refer to Land Ownership Procedure for manual transfer instructions.

Reference: LOAN Property Task Procedure; Completing or Removing Property Task
Input: Verified property with all references correct
Process: Execute completion or cancellation
Decision: Proceed or cancel?
Output: Task completed or cancelled; route to next stage

Procedure Summary

  • 5-step process (Q1-Q5) for processing LOAN Property Tasks
  • Q1: Identify task type (Resolve Match/Conflict, Multi Property, High Volume, Parcel)
  • Q2: Search for property using Find Property, Muni Returns, COI, or Find Seller's Properties
  • Q3: Verify title references are correct and complete; handle Part Indicator for shared references
  • Q4: Update property address, references, and possession date as needed
  • Q5: Complete or cancel Property Task
  • Reference/Address score thresholds: Very High/High=link; Medium=verify; Low/Poor=reject
  • Site Valuation Request SLA: 1-2 weeks typical response
