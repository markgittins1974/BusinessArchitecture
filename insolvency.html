Q1 - Has valid insolvency correspondence been received?

  • Open insolvency action item from work tray (filter: Work Item Description = Insolvency, Revenue Line = Land Tax) and review IP correspondence from CM10.
  • If no action item exists and correspondence received directly from SRO DMS/Legal Team, create action item and save to CM10.
  • Check relevancy: if IP documentation is older than 6 months, verify customer is still insolvent via ASIC lookup or Equifax/VEDA check.
  • Determine whether IP request relates to Land Tax, Payroll Tax, or both:
    • Land Tax only: follow this procedure.
    • Payroll Tax only: refer to PTX procedures (Initial Checks, Payroll Tax Node instructions).
    • Both: complete LTX part and email ptx.capture@sro.vic.gov.au advising customer number and PTX insolvency.
  • If IP document lists multiple entities/properties: review each entity/property one at a time, send separate correspondence for each, complete separate action items for each.
  • Allocate action item to your name immediately. Change status from PENDING ACTION to IN PROGRESS.
Do not proceed if correspondence is not current or relevant.
Procedure reference: Reviewing IP Correspondence (p.7, 11-13); Before you begin (p.6); IP Notes for Consideration (p.12-13)
Input data:
  • IP correspondence (from CM10 or direct email from DMS/Legal)
  • Entity name (Individual / Company / Trust/Trustee)
  • IP name and representative persons
  • Appointment type and date
  • Properties listed / referenced
  • Contact details of IP
  • Advice on sale of any land
  • Age of IP documentation (6-month relevancy threshold)
Processing responsibility:
  • LRA staff - review correspondence, create/allocate action item
  • e-Sys - action item creation, work tray management
  • CM10 - correspondence storage
  • ASIC / Equifax VEDA - insolvency currency verification
Decision: Intake routing:
  • LTX only - proceed to Q2
  • PTX only - refer to PTX procedures
  • Both LTX and PTX - complete LTX, email ptx.capture@sro.vic.gov.au
  • Not current/relevant - update and complete action item
Output:
  • Action item created/allocated and set to IN PROGRESS
  • IP correspondence reviewed and validated
  • Revenue line routing determined
Iteration constraint: Multiple entities/properties in one IP document must be processed one at a time with separate correspondence and action items for each.

Q2 - Is the customer registered in e-Sys?

  • Search for customer using identifiers from IP correspondence.
  • If not found, escalate search:
    • Check IP correspondence for correct entity, confirm Name and/or ACN/ABN.
    • Complete ABN Lookup (if relevant).
    • Complete ASIC search.
    • Complete Equifax / IQ Connect (VEDA) search.
  • If still cannot find customer: send clearance correspondence using template "LTX - NOT REGISTERED IN E-SYS - CLEARANCE CORRESPONDENCE".
  • Save incoming and outgoing correspondence to CM10 folder VF/24/41 under current year with naming conventions:
    • Individual: Surname - Insolvency with no Associated Customer - Received/Sent - Date
    • Company: ACN/ABN (no spaces) - Insolvency with no Associated Customer - Received/Sent - Date
Procedure reference: Customer Not Registered in E-Sys (p.7); Customer Is Registered in E-Sys (p.8)
Input data:
  • Customer identifiers from IP correspondence (name, ACN, ABN)
  • e-Sys search results
  • ABN Lookup results
  • ASIC search results
  • Equifax/VEDA search results
Processing responsibility:
  • LRA staff - search escalation
  • e-Sys - customer lookup
  • ABN Lookup / ASIC / Equifax VEDA - external verification
  • CM10 - correspondence filing (folder VF/24/41)
Decision: Registration status:
  • Registered - proceed to Q3
  • Not registered (after 4-step search) - issue clearance correspondence, close
Output:
  • Customer confirmed in e-Sys - proceed
  • Or clearance correspondence issued and filed to CM10

Q3 - Does the customer own land or have outstanding land tax debt?

  • Review customer tree for: Investigation Node, Payroll Tax Node, IP Manager Node, Land Owner node.
  • If customer is a trustee: check if the trust owns properties or if there are multiple trusts. Load associated Trustee/Trust accounts into Customer Tree. Use ASIC/COI/LTO reports to identify land ownership.
  • Check Land Owner node for current property ownership (MC322 screen).
  • If property listed: note property and continue.
  • If no property listed: check for outstanding debt. If outstanding debt exists, POD will need to be created.
  • If no outstanding debt and no land ownership: use correspondence "LTX REGISTERED - WAS A LANDOWNER / HAS NO DEBT".
  • Check insolvency application via Content Manager (TRIM): search by customer number, view correspondence in LTX folder.
Privacy - CRITICAL: Only release information directly related to the IP request. If IP-provided information conflicts with e-Sys (names, DOB, addresses, property ownership especially around Trustees and Trusts), send only relevant correspondence. Mortgagee in Possession: only release property-specific information.
Do not issue a Proof of Debt where land is currently owned - SRO is a secured creditor.
Procedure reference: Initial Insolvent Customer Check (p.9-11); Privacy Checks (p.13); Checking Insolvency Application via CM/TRIM (p.11)
Input data:
  • Customer tree nodes: Investigation, Payroll Tax, IP Manager, Land Owner
  • Land ownership data (MC322 - Properties Owned by Customer)
  • Outstanding liabilities
  • Trust/Trustee account relationships (multiple possible)
  • ASIC / COI / LTO reports (for ownership verification)
  • TRIM/CM10 records (IP correspondence)
Processing responsibility:
  • LRA staff - tree review, ownership assessment, privacy checks
  • e-Sys - customer tree, MC322 screen
  • TRIM/CM10 - correspondence retrieval
  • ASIC / COI / LTO - ownership verification
Decision: Ownership and debt status (four binary variables):
  • Currently owns land: Yes / No
  • Has outstanding debt: Yes / No
  • Bankrupt Estate: Yes / No
  • Customer no longer insolvent: Yes / No
Output:
  • Ownership and debt status determined - feeds Q5 and Q6 correspondence selection
  • Trust/Trustee relationships mapped
  • If no longer insolvent: add cessation date, remove IP names, revert mailing protocols, check ASIC for current address
Trust complexity: Trust account can have multiple Trustee accounts AND a Trustee can have multiple Trust accounts. Land ownership must be confirmed for each Trustee and Trust account separately.

Q4 - What insolvency appointment type applies?

  • Identify IP appointment type from correspondence. Seven types:
    • Receiver and Manager
    • Controller
    • Administrator
    • Liquidator
    • Deed Administrator (Deed of Arrangement)
    • Bankruptcy
    • Mortgagee of Possession - NOT an insolvency. IP is not added. Only property-specific information released.
  • Check (IP) Managed Customer Node in customer tree. If listed, double-click to view IP Managers window - check relationship and effective dates.
  • If IP matches correspondence: proceed to Update Land Taxpayer Mailing Protocols.
  • If IP does not match: go to Adding an IP Insolvency Type instructions.
  • If no (IP) Managed Customer Node: look up IP in Insolvency Practitioner Spreadsheet (One Note Templates, Insolvency TAB). Copy CID, search in e-Sys, add to tree.
  • If IP not in spreadsheet: search e-Sys by name/ACN/ABN. If still not found, create new IP customer (refer First Time Liability instructions, BTO if needed). Add new IP to spreadsheet.
  • Adding IP type is a two-part process: adds IP type to IP Customer Node AND adds IP to Insolvent Customer.
  • For trustee insolvencies: IP appointment must be added to both Trustee and Trust accounts. Update mailing protocols on both.
  • For joint ownership: update individual Land Taxpayer Mailing Protocol to IP address. For joint ownership accounts, only update if Lead customer is the insolvent.
  • Update mailing protocols: Land Taxpayer node, Contact tab, Protocols tab (Line 1: Entity Name with Appointment type, Line 2: C/- IP Name, Line 3: ATTN: IP appointed person).
  • For bankrupt estates: manually update customer name with "(Bankrupt Estate)" beside surname.
  • Cessation: cease date = one day prior to next appointment. Exception: Receivers and Managers may continue during additional appointments.
Do not release information beyond the scope of the appointment. Mortgagee of Possession is NOT an insolvency.
Procedure reference: Insolvency Practitioner - IP (p.14-20); Adding an IP Insolvency Type (p.18-19); No IP Managed Customer Node (p.15); Update Land Taxpayer Mailing Protocols (p.21-23); Cessation Date (p.21); Customer No Longer Insolvent (p.20)
Input data:
  • IP appointment type (7 types, Mortgagee of Possession excluded as non-insolvency)
  • Appointment effective date and cessation date
  • (IP) Managed Customer Node in e-Sys
  • IP Listing Spreadsheet (One Note, Insolvency TAB)
  • Trust/Trustee account relationships
  • Joint ownership status (Lead customer identification)
Processing responsibility:
  • LRA staff - IP verification, appointment setup, mailing protocol updates
  • e-Sys - IP Customer node, Insolvency Practitioner node, Mailing Protocols (Contact/Protocols tabs), customer name updates
  • IP Listing Spreadsheet - lookup and maintenance
  • BTO - assistance for new IP customer creation if needed
Decision: Appointment classification and IP setup:
  • IP exists in system and matches - update mailing protocols
  • IP exists but does not match - add new appointment type (two-part process)
  • IP not in system - create new IP customer, add to spreadsheet
  • Mortgagee of Possession - do not add IP, release property-specific info only
  • Multiple concurrent IPs (e.g. R&M + Liquidator) - add all, mailing stays with R&M
Output:
  • IP linked to insolvent customer in e-Sys (both IP node and customer node)
  • Mailing protocols updated (Line 1/2/3 format)
  • Customer name updated for Bankrupt Estates
  • IP Listing Spreadsheet updated (if new IP)
  • Trust and Trustee accounts both updated (if applicable)

Q5 - Is the debt pre-appointment, post-appointment, or both?

  • Pre-appointment debt: Any liability outstanding prior to and including the assessment year where debt became payable as at midnight 31 December before the appointment date. Not apportioned for land tax - advised for the full year. POD always matches pre-appointment debt.
  • Post-appointment debt: Any outstanding assessment on or after 31 December after the appointment date.
  • Example: IP appointed June 2023. Pre-appointment = liabilities up to and including 2023 year (payable as at 31 Dec 2022). Post-appointment = assessments from 31 Dec 2023 onwards.
  • Navigate to Land Taxpayer node, double-click Liabilities. Assessment Year Liability Summary opens - scroll right for Net Liability column. Note date, amount, interest, penalties.
  • For complex calculations: use POD Reconciliation Template (One Note Templates, Insolvency TAB).
  • Check suspense: Land Taxpayer node, Allocations, tick Suspense, click Apply. Allocate suspense to outstanding liabilities. If further suspense exists, contact IP for banking details.
Ensure all assessments for pre and post-appointment debt have been issued before sending correspondence.
Procedure reference: Determining Pre-Appointment and Post-Appointment Debt (p.26-28); Suspense Money (p.28-29)
Input data:
  • IP appointment date (from Q4)
  • Assessment Year Liability Summary (Net Liability, interest, penalties per year)
  • Assessment issuance status (all must be issued before correspondence)
  • Suspense/allocation records
  • POD Reconciliation Template (for complex calculations)
Processing responsibility:
  • LRA staff - debt classification, suspense review
  • e-Sys - Liabilities node, Allocations screen
Decision: Debt timing classification:
  • Pre-appointment debt only
  • Post-appointment debt only
  • Both pre and post-appointment debt
  • No debt
Output:
  • Debt classified by timing relative to appointment date
  • Amounts noted (liability, interest, penalties per year)
  • Suspense allocated to outstanding liabilities
  • All assessments confirmed as issued
  • Feeds Q6 correspondence template selection

Q6 - What correspondence must be issued based on ownership and debt status?

  • Select the correct correspondence template from One Note Templates (Insolvency TAB). Template selection is driven by four factors: registered status, land ownership, debt status, and bankrupt estate status.
  • 12 correspondence templates:
    1. Not Registered in E-Sys - Clearance
    2. Registered - Information Does Not Match
    3. Registered - Was a Landowner, No Debt
    4. Registered - Landowner, No Debt
    5. Registered - Landowner with Pre-appointment Debt Only (+ LTX Charge)
    6. Registered - Landowner with Post-appointment Debt Only (+ LTX Charge)
    7. Registered - Landowner with Pre and Post-appointment Debt (+ LTX Charge)
    8. Registered - No Longer Owns Land, Has Debt - POD
    9. Registered - Bankrupt Estate, Was Landowner, No Debt (+ LTX Charge)
    10. Registered - Bankrupt Estate, Was Landowner, Has Debt - POD
    11. Registered - Bankrupt Estate, Landowner, Has Debt - POD (+ LTX Charge)
    12. Registered - Bankrupt Estate, Landowner, No Debt (+ LTX Charge)
  • Debt letter: When land is still owned and there is debt, SRO is a secured creditor. Issue debt letter and advise IP to request clearance certificate via LTX Express for further liability details.
  • Land Tax Charge: Check Recovery Node, PropChg box. If ticked and single property = NFAR. If ticked and multiple properties = continue. If not ticked = create "LTX CHARGE REVIEW REQUIRED" action item with comments (IP type, assessment year, trust/trustee CIDs, property name/LID, what is required).
  • Proof of Debt (POD): Issued when land already sold and debt remains, OR for Bankrupt Estates. Two templates: Bankruptcy Estate POD (Individual) and General POD (Company/Trusts). Go to One Note Templates, Insolvency tab, select template, click Word-POD, edit, save as PDF, attach to correspondence.
  • Joint ownership POD: Issued under joint owner showing all owners. If bankrupt person also has individual account debt, complete additional POD for individual account.
Do not issue POD where land is currently owned and SRO is a secured creditor. Issue a debt letter instead.
Procedure reference: Correspondence for IP (p.32-33); Create Proof of Debt (p.30-31); LTX Charge Review (p.29-30); Introduction/Background (p.2-3)
Input data:
  • Registration status (from Q2)
  • Land ownership status (from Q3): currently owns / was owner / never owned
  • Debt status (from Q5): pre-appointment / post-appointment / both / none
  • Bankrupt Estate status (from Q4)
  • Recovery Node - PropChg box (ticked/not ticked)
  • Number of properties owned (single vs multiple - affects LTX Charge path)
  • Joint ownership status (affects POD issuance)
Processing responsibility:
  • LRA staff - template selection, POD creation, LTX Charge review
  • e-Sys - Recovery Node check, LTX Charge action item creation
  • One Note Templates (Insolvency TAB) - correspondence and POD templates
  • TRIM/CM10 - correspondence storage
Decision: Correspondence type (driven by 4 binary inputs = 12 templates):
  • Registered (Y/N)
  • Currently owns land (Y/N)
  • Has debt (Y/N) and timing (pre/post/both)
  • Bankrupt Estate (Y/N)
Output:
  • Correct correspondence template selected and issued
  • Debt letter issued (if landowner with debt - SRO secured creditor)
  • POD created and attached (if no land + debt, or Bankrupt Estate)
  • LTX Charge action item created (if PropChg not ticked)
  • Correspondence saved to TRIM/CM10
High-cardinality branching: 12 distinct correspondence templates driven by 4 binary decision variables. This is the most complex decision point in the procedure.

Procedure Summary

  • Covers identification and processing of land tax insolvency matters for registered and unregistered customers.
  • Addresses review of IP correspondence and verification of insolvency status.
  • Determines land ownership status and existence of outstanding land tax liabilities.
  • Distinguishes pre-appointment and post-appointment land tax debt.
  • Specifies issuance of clearance correspondence, debt letters, Proof of Debt documents, and land tax charges.
  • Time standard: 20 minutes per work item.

Correspondence Template Matrix (12 templates)

Templates located in One Note Templates - Insolvency TAB. Selection driven by 4 binary variables.

  1. Not Registered - Clearance
  2. Registered - Information Does Not Match
  3. Registered - Was Landowner, No Debt
  4. Registered - Landowner, No Debt
  5. Registered - Landowner, Pre-appointment Debt (+ Charge)
  6. Registered - Landowner, Post-appointment Debt (+ Charge)
  7. Registered - Landowner, Pre + Post Debt (+ Charge)
  8. Registered - No Land, Has Debt - POD
  9. Bankrupt Estate - Was Landowner, No Debt (+ Charge)
  10. Bankrupt Estate - Was Landowner, Has Debt - POD
  11. Bankrupt Estate - Landowner, Has Debt - POD (+ Charge)
  12. Bankrupt Estate - Landowner, No Debt (+ Charge)

Appointment Types (7)

  • Receiver and Manager: Takes control of company undertaking/property for creditor enforcement
  • Controller: Controls particular property for charge enforcement
  • Administrator: Manages business, disposes assets, restructures for creditor returns
  • Liquidator: Winds up company, collects and distributes assets
  • Deed Administrator: Manages Deed of Company Arrangement (DOCA)
  • Bankruptcy: Trustee manages bankrupt estate assets and creditor payments
  • Mortgagee of Possession: NOT an insolvency - IP acts for mortgagee, property-specific info only

Structural Limitations and Ambiguities

Correspondence classification: 12 templates with overlapping conditions driven by 4 binary variables. High decision complexity - may benefit from a decision table or rules engine.
Appointment handling: Multiple concurrent appointment types (e.g. R&M + Liquidator) increase branching. Mailing protocol rules differ by combination.
Debt timing: Dependency on assessment issuance dates relative to appointment date. Pre-appointment debt is not apportioned but advised for the full year.
Trust/Trustee complexity: Many-to-many relationships between trusts and trustees create non-linear branching not easily proceduralised.
Privacy constraints: Information release is scoped to IP appointment authority. Conflicts between IP-provided data and e-Sys records require manual judgement.
