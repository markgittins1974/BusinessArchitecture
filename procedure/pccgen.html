Property Clearance Certificate - Procedure Overview

Purpose

Guide Customer Service Officers (CSOs) to identify correct property/vendor, process Property Inquiry Applications (PIAs), and generate Property Clearance Certificates (PCCs) consistently and accurately.

What is a Property Clearance Certificate?

  • PCC Definition: Details the amount of tax secured by a charge on land
  • What it includes: Land tax owed for current year + outstanding liabilities from prior years + Windfall Gains Tax (WGT) + Commercial & Industrial Property Tax (CIPT) + Vacant Residential Land Tax (VRLT) + deferral interest
  • NOT a Tax Assessment: It's a "snapshot" of liabilities on customer's file at time of issuance
  • Right Person/Right Property Rule: Ensure certificate reflects correct property and correct person/entity

Key Facts

  • ~300,000 PCC applications received annually via e-Business
  • Fee: $18.90 (increases 1.16 fee units on 1 July annually)
  • Turnaround time: 10 working days (24 hours if no issues hold-up)
  • Reissues allowed: Unlimited within 90 days of original issuance
  • Automation applies to many applications via e-Sys rules
  • Manual intervention required for complex cases with warnings

Time Standard: 6 minutes per item

Key Principle: Ensure certificate reflects "right person/right property" — correct property identified and correct owner/entity on certificate
Input: Property Inquiry Application (PIA) via e-Business, with property/vendor/owner details
Process: Q1-Q5: Launch → Validate → Process → Generate → Resolve
Decision: Is property correct? Is vendor correct? Are warnings resolved? Pro rata required?
Output: Property Clearance Certificate issued or manual intervention required

Q1 - Launch PIA Work Item

Accessing PIA from Workflow

  1. Open PIA work item from CSO workflow tray
  2. Review Details Tab for application information
  3. Verify source: e-Business (registered/non-registered user), agency (Landata, Dye & Durham, Infotrack)
  4. Note applicant name and application reference
  5. Identify vendor (owner) on application

Understanding PIA Sources

  • e-Business applicants: Direct SRO website submissions (registered or non-registered)
  • Agency applications: Landata, Dye & Durham (Anstat), Infotrack lodge on behalf of applicants
  • Details tab: Shows applicant name/email, address, reference number

Critical Information to Extract

  • Property address and references (from "Property Search" section)
  • Vendor name (owner)
  • Applicant requesting certificate (may be purchaser, mortgagee, agent)
  • Any property clarification already noted
Note: If PIA shows property location is not rateable (e.g., Crown land, exempt property), return to applicant for clarification or reject application.
Input: PIA work item in CSO workflow tray
Process: CSO launches item, reviews Details tab and property references
Decision: Is property information visible and identifiable?
Output: PIA details extracted, ready for Q2

Q2 - Validate Property and Vendor

Property Validation Process

The flowchart process requires confirming:

  1. Property Search: Search for property by references (plan number, LID, address)
  2. Match Assessment: Do first two property references match the property and owner on application?
  3. Clarification Required: If mismatch, contact applicant to clarify vendor name and property details

Three Possible Outcomes:

  • Both first two references match: Proceed to vendor clarification
  • Only one of first two references matches: Contact applicant for clarification and edit as requested
  • None of references match: Contact applicant for clarification and edit as requested

Vendor Clarification

Acceptable vendors for PCC:

  • Current Registered Proprietor on title
  • Purchaser (must have Current Registered Proprietor name on application)
  • Mortgagee in possession (e.g., Major Bank)
  • Power of Attorney
  • Executor of deceased estate
  • On-seller
  • Company Director & Crown Legal Interests

For non-matching vendors, add comment to PIA: "Vendor is current proprietor" or clarify with applicant.

CRITICAL: Certificate cannot be issued if vendor does not match one of the acceptable categories above. If vendor is not identifiable or not acceptable, redirect PIA to Manual Certificate Tray or request clarification.
Procedure Detail: Property Search Methods▼

Searching by Property References

In e-Sys, use Search tools:

  • Plan number: Search for plans folder, request copy from applicant
  • LID: Land Identification number (unique property identifier)
  • Address: Street address and suburb
  • COI (Compliance Operational Intelligence): Title search for vendor details

Handling Missing or Unclear References

If references missing, options:

  • Request from applicant
  • Search new plans folder if property is newly subdivided
  • Refer to Senior CSO if unsure
Input: Property details from PIA (address, references, vendor name)
Process: Search e-Sys for property, verify vendor matches acceptable category
Decision: Do property references match? Is vendor acceptable?
Output: Property/vendor validated OR clarification required

Q3 - Process Pro Rata and Area Entitlement

Pro Rata - Area or Unit Entitlement Decision

Determine whether property is part of subdivided land requiring pro rata calculations:

Three Scenarios:

  • Scenario 1: Property in Parcel — Part of subdivided land with multiple properties (pro rata required)
  • Scenario 2: No Child Area/Unit Entitlement — Issue certificate on parent property with comment: "Certificate has been issued on the parent property"
  • Scenario 3: Child Properties with Unit Entitlements — Issue certificate without pro rata but add comment: "Certificate has been issued on the parent property"

60/40 Rule for Pro Rata Scenarios

If new plan has roads or common property:

  • Apply 60/40 rule: Parent Area Discount factors in roads/common property so tax applies when all child properties sold
  • Not all scenarios use 60/40: Check plan design first
  • Examples provided: When 60/40 applies vs. when it doesn't

Pro Rata Calculation Process

  1. Determine parent property liabilities
  2. Calculate pro rata based on area/unit entitlement
  3. Select area for pro rata (must use area as if available)
  4. Select parent property and enter child area
  5. Check other sources for area (electronic plan, other certificates)
  6. Can area be found? If yes, continue; if no, escalate
  7. Click OK for pro rata calculation
  8. If multiple parent properties with different liability dates, perform separate pro rata for each
  9. Continue online PIA process and issue certificate
Complex Areas: If property appears in parcel, plan has complex structure, or pro rata seems ambiguous, escalate to senior staff. Accuracy critical for pro rata calculations.
Input: Property details, plan information, area measurements
Process: Identify if pro rata required, apply 60/40 rule if applicable, calculate area/unit entitlement split
Decision: Is pro rata needed? Is 60/40 rule applicable? Are area calculations correct?
Output: Pro rata applied (or not), certificate ready for issuance with correct liability split

Q4 - Generate and Issue Certificate

Copy of Plan Requirements

Full copy of plan required when property is in e-Sys. Process:

  1. Check if property located in e-Sys: YES → Load property to tree
  2. If NO → Search for property using references (plan number, address)
  3. Attempt to locate in "temporary folder"
  4. If temporary folder not open, select "Search New Plans Folder"
  5. Right-click plan number (field) and select "view electronic plan"
  6. If did temporary folder exist? If NO, search new plans
  7. Was plan located? If NO, contact applicant for plan copy or refer to escalation
  8. If plan located, copy to PIA tree
  9. Add note to PIA: "Requested copy of plan, (request date) and use LID"

Certificate Generation

  1. Once property/vendor/pro rata validated, system can auto-generate
  2. If all details clear and matching, click "Generate B97 Certificate Button"
  3. Warning or message appears: Review and resolve before issuance (see Q5)

Automation vs. Manual Processing

  • Auto-issue (24 hours): Application matches e-Sys records, no objections, nothing holding up file
  • Manual (requires CSO action): Warnings exist, vendor clarification needed, property hold-ups, complex cases
Input: Validated property/vendor, pro rata calculated, plan copy located
Process: Generate certificate in e-Sys, system produces B97 Certificate number
Decision: Are all warnings resolved? Is certificate ready to issue?
Output: Certificate generated with reference number

Q5 - Resolve Warnings and Complete

PIA Warning Flowchart

When certificate generation triggers warnings, CSO must resolve before issuance. Warning types:

Warning Categories:

  • Property Warning: Property details mismatched or unclear
  • Vendor Warning: Vendor name doesn't match title or is not acceptable
  • Liability Warning: Outstanding liabilities, zero value, or recovery issues
  • Other Warnings: Settlement dates, commissions, duplicate issues, customer warnings

Common Manual Intervention Cases

Vendor Clarification (See Q2 for Details)

  • Contact applicant for clarification on vendor identity
  • Request written authority if vendor is not current proprietor
  • Verify vendor on COI or title documents
  • Update application once clarified

Liability or Recovery Issues

  • Zero value on property: If vendor is statutory body (e.g., Director of Housing), may override warning. Otherwise escalate.
  • Write off required: Liability <$50 and economically unviable to chase — request manual certificate or refer to management
  • Recovery non-issue: Liability ceased prior to 31 Dec of prior year — manual certificate required

Change of Ownership

  • Owner Unknown: Vendor doesn't match e-Sys — escalate to Land Ownership tray for BTO action
  • COI Issues: Vendor match check via Volume/Folio or title details
  • Dates impact: If vendor is registered proprietor before/on 31 Dec of current year — process normally. If after, may impact assessment year.
  • Estates: Check if property has transmission application; if yes and dealing type is transmission, may need Death or Estate tray action

Escalations and Manual Certificates

Cases requiring escalation or manual certificate:

  • Vendor not acceptable for PCC
  • Complex ownership (trusts, multiple parties, deceased estates)
  • Liability or write-off issues
  • PIA has open action items (Land Ownership, Assessing escalations)
  • Property parcel or subdivision complexities
CRITICAL: Never issue certificate with unresolved warnings unless explicitly documented in application notes and approved by Senior CSO. Each warning must be addressed.
Procedure Detail: Requesting Further Clarification▼

Agency Clarification Request

For applications via Landata, Dye & Durham, Infotrack:

  • Send email to relevant agency with application number and User ID
  • Specify vendor name clarification needed or property details required
  • Note: Agencies may have multiple hours to respond

e-Business Applicant Clarification

For direct website submissions:

  • Contact applicant directly via phone if number available
  • Ask to clarify vendor name and/or property details
  • If multiple attempts unsuccessful, send email using template
  • Place note in PIA: "Vendor Clarification requested, [date] and user ID"
  • Place PIA in reserve for 2 weeks
  • If no reply within 2 weeks, complete item without making requested changes

Property Clarification

If first two property references don't match property and owner:

  • Contact applicant
  • Request clarification of which property references correct
  • Edit application once applicant confirms
  • Proceed to certificate generation
Input: Generated certificate with warnings, unresolved issues
Process: Review each warning, resolve through clarification/escalation/override/manual certificate
Decision: Can warning be resolved? Does it require escalation? Is manual certificate needed?
Output: Certificate issued OR escalated to appropriate tray/manual process

Pro Rata Calculation Rules

When Pro Rata is Required

Property is part of subdivided land (parcel) with multiple child properties. Liabilities must be split proportionally by area or unit entitlement.

60/40 Rule (Parent Area Discount)

Apply when new plan of subdivision has roads or common property:

  • Calculation factors in roads and common property
  • When all child properties sold, land tax applies to roads/common property
  • Not all subdivisions use 60/40: Check plan design
  • Examples: Plans with clear roads/common property apply rule; isolated properties don't

Unit Entitlement (Apartments/Units)

If property has unit entitlements (e.g., apartments in building):

  • Issue certificate on parent property with comment: "Certificate has been issued on the parent property"
  • If child properties with unit entitlements and other child properties with square meters, issue without pro rata but add comment

Pro Rata Calculation Steps

  1. Identify parent property liabilities
  2. Calculate child's share: (Child area / Parent area) × Parent liability
  3. Check electronic plans or certificates for area information
  4. If area cannot be found, escalate to senior CSO
  5. Enter child area and parent property into pro rata calculation
  6. Click OK; system calculates split
  7. If multiple parents with different liability dates, perform separate pro rata for each

Vendor Validation & Acceptable Vendors

Acceptable Vendors for PCC

  • Current Registered Proprietor on title
  • Purchaser (must have Current Registered Proprietor name on application)
  • Mortgagee in possession (e.g., Major Bank)
  • Power of Attorney (legally appointed)
  • Executor of deceased estate (with probate)
  • On-seller
  • Company Director & Crown Legal Interests

Vendor Clarification Required

If vendor does NOT match current proprietor or acceptable category:

  • Contact applicant (phone or email)
  • Request clarification of vendor identity
  • Confirm vendor is acceptable for PCC
  • Update PIA application once clarified
  • If applicant cannot verify, cannot issue certificate

Non-Acceptable Vendors

If vendor is real estate agent, financial advisor, or other non-acceptable party:

  • Cannot issue certificate
  • Redirect PIA to Cancellation Tray or Manual Certificate Tray
  • Notify applicant of requirement

Vendor Protected by Prior Certificate

Special circumstance: Vendor is protected from prior year tax liabilities if they purchased with prior year certificate.

  • Request copy of original certificate
  • Confirm payment received for prior certificate liability
  • Include prior years on new certificate (or note which years covered by prior cert)

  • Automation: Applications with no issues auto-issue in 24 hours; others require manual action
  • Warnings: Must resolve before issuance (property, vendor, liability, settlement, duplicates, etc.)
  • Pro Rata: Required for subdivided land; 60/40 rule applies when roads/common property present
  • Escalations: Complex cases, vendor issues, write-offs, ownership disputes go to appropriate tray
