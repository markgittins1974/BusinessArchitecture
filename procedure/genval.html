General Valuations - Administration Procedure Overview

Purpose

Provide instruction on how to record, store, register, and report on certified general revaluation data received from the Valuer General Victoria (VGV).

Scope

  • Recording VGV advice on receipt of general revaluations
  • Storing data in appropriate network locations for processing
  • Registering general revaluations in the administration register
  • Tracking file status through formatting, testing, QA, and production load
  • Reporting on revaluation progress via end of month reports

Scope Exclusions

  • Does NOT explain how to use revaluation data (i.e., load to Production)
  • Does NOT cover all exceptional circumstances (use discretion)
  • Related procedures: Formatting, Testing, Notes and Anecdotes procedures

User Assumptions

  • User has access to Valuations inbox (Outlook)
  • User has access to S: network drive locations
  • User has basic Excel skills
  • User has awareness of overall valuations process

Key Annual Cycle

Every year, Victoria's Valuation Authority undertakes general revaluations on all rateable and non-rateable leviable land within their municipality. Valuation authority submits data to VGV for certification, then VGV forwards certified data to SRO via email for recording and processing.

Q1 - How to Receive and Record VGV Certified Data?

Receiving Process

  1. Navigate to the Valuations inbox folder in Outlook
  2. Open the email titled '20XX General Valuation Certified Data'
  3. Note: Subject line contains important information

Email Subject Line Format (Legend)

Format: [Valuation Level] – [Email Type] – [Municipality] – [VGV Reference]

Example: Level 2022 – Type 1 – Ballarat – VGV-2022-BAL-001

Recording Receipt Date

  1. Note the date the VGV email was RECEIVED in the Valuations inbox
  2. Open the '20XX General Valuation Register' spreadsheet located at:
    H:\LandTax\Common\Land_tax\Valuations Team\Administration\General Reval 20XX
  3. Record the received date in the register

Counting Properties

To determine the number of properties:

  1. Open the Type 1 file received in the email (unzip the file)
  2. Count the number of individual lines in the file
  3. Record this count in the 'Number of properties' field in the register
CRITICAL: The date recorded is the date the VGV email was received, NOT the date the data was processed or uploaded.
Procedure Detail: Email Subject Line Breakdown & File Types▼

Email Subject Line Components

Valuation Level: The assessment year (e.g., Level 2022, Level 2023)

Email Type: Type 1 = Full data set, Type 2 = Amendment/Update

Municipality: Council name (e.g., Ballarat, Melbourne, Geelong)

VGV Reference: Unique reference number assigned by VGV

File Types in Attachment

  • Type 1 File: Complete revaluation data for the municipality
  • Type 2 File: Amendment file with updates/corrections
  • Files are compressed in a ZIP archive within the email
  • Unzip to access individual text files

Register Location Reference

General Valuation Register location:

H:\LandTax\Common\Land_tax\Valuations Team\Administration\General Reval 20XX

  • Replace '20XX' with actual year (e.g., General Reval 2023)
  • File contains 'Register-Format and test' tab for data entry
  • File contains 'Reval schedule' tab for progress tracking
  • File contains 'Burndown chart' tab for reporting
Input: VGV email with subject '20XX General Valuation Certified Data'
Process: Open email, record receipt date, count properties from Type 1 file
Decision: Email format correct? Receipt date recorded?
Output: Receipt date and property count recorded in General Valuation Register

Q2 - Where and How Should Files Be Stored?

File Storage Locations

Files must be stored in TWO locations: S: network drive AND TRIM document management system.

Location 1: S: Network Drive

Path: S:\FTP\Val_Returns\Level XXXX\[Municipality]\received

Example: S:\FTP\Val_Returns\Level 2022\Ballarat\received

Step-by-Step Storage Process

  1. Open the ZIP file attached to the VGV email
  2. Select all relevant text files
  3. Locate the correct council file within the correct valuation level folder on S: drive
  4. Example structure: S:\FTP\Val_Returns\Level 2022\Ballarat
  5. CREATE a new folder titled 'received' inside the council folder
  6. Copy and paste the files into the 'received' folder
  7. Verify files successfully copied (check file count and sizes)

Location 2: TRIM Document Management

  1. Once files saved to S: drive, save the EMAIL (not the files) to TRIM
  2. Save email to relevant council folder in TRIM
  3. Example: TRIM\Valuations\2022\Ballarat\General Revaluation

Email Folder Management (Outlook)

  1. After saving email to TRIM, move email from 'Revaluation Data' folder to 'completed' folder in Outlook
  2. This marks the receipt process as complete
CRITICAL: Files go to S: drive 'received' folder. EMAIL goes to TRIM. Do NOT save files to TRIM, only the email.
Folder Structure Required: S:\FTP\Val_Returns\Level [YEAR]\[MUNICIPALITY]\received\ [VGV data files go here]

Decision: Correct level folder? Correct municipality? Received folder created?

Q3 - How to Track Formatting, Testing, and QA Progress?

Register Table Updates

As files are formatted, tested, QA checked, and loaded to production, update the 'Reval schedule' tab of the General Valuation Register with the following information:

Register Columns and Data Entry

Column Name Data to Enter Completed By
Municipality Council name Pre-loaded to register
Date Received Date VGV email received in Valuations inbox Formula (from Register-Format tab)
File Status Formatted OR Tested Formula (from Register-Format tab)
Week Assigned Date file assigned for formatting/testing Grade 3 co-ordinating revaluation
Assigned for QA Name of officer allocated for QA check Grade 3 co-ordinating revaluation
QA Completed Yes/No QA officer
Final Sign Off Name of officer for final approval BTO or Team Leader
Number of Properties Count of properties to be updated Formatting officer
2022 Production Run Time Time file took to run previous year (indicator) Pre-loaded to register
Load Week Week file planned to be loaded to production Grade 3 co-ordinating revaluation
Loaded Date file was loaded to production Grade 3 co-ordinating revaluation

Production Load Planning

Grade 3 co-ordinating the revaluation cycle uses the space at bottom of 'Reval schedule' tab to plan weekly loading:

  • Typical load: 3-4 files loaded each day
  • Maximum run time: 45 minutes per load cycle
  • Schedule accounts for system capacity and overnight processing

Procedure Detail: Register Tab Structure & Formula Examples▼

General Valuation Register Sheet Tabs

  • 'Register-Format and test' tab: Data entry point for formatting/testing progress
  • 'Reval schedule' tab: Summary table with formulas pulling from Register-Format tab
  • 'Burndown chart' tab: Graph showing progress for monthly reporting

Formula Fields

The following columns use Excel formulas to pull data automatically:

  • Date Received: Formula pulls from 'Register-Format and test' tab
  • File Status: Formula pulls Formatted/Tested status from 'Register-Format and test' tab

Manual Entry Responsibilities

Grade 3 co-ordinating revaluation manually enters:

  • Week Assigned (date formatting work begins)
  • Assigned for QA (officer name)
  • Load Week (planned week for production)
  • Loaded (actual date loaded to production)

Production Run Time Reference

Column '2022 Production Run Time' shows prior year run time as indicator for planning. Use this to estimate scheduling and system load capacity.

Input: File formatting/testing completion, QA results, production load date
Process: Update Reval schedule table with status, dates, officer names
Decision: All required fields entered? Status correct? Dates logical?
Output: Register updated, progress tracked, ready for reporting

Q4 - How to Report Revaluation Progress in Monthly Reports?

End of Month Report Content

Information is provided to LGA team leader and LRB branch manager through the end of month report to provide a status update of general revaluation files.

Data Source

Open the General Valuation Register:

H:\LandTax\Common\Land_tax\Valuations Team\Administration\General Reval 20XX

Report Components

  1. Burndown Chart Graph - From 'Burndown chart' tab
    • Visual representation of files loaded vs. remaining
    • Included in end of month report
    • Shows progress trajectory
  2. Summary Text
    • Number of files loaded (completed)
    • Number of files remaining (to be loaded)
    • Overall progress percentage
    • Expected completion date

Report Delivery

  • Report delivered to LGA Team Leader
  • Report delivered to LRB Branch Manager
  • Timing: End of each calendar month
  • Includes snapshot of revaluation load status at time of reporting

Data Extraction Process

  1. Open General Revaluation Register (year-specific file)
  2. Go to 'Burndown chart' tab
  3. Copy the graph image
  4. Include in monthly report document
  5. Compile text summary from Reval schedule table (count files loaded vs. remaining)
  6. Provide narrative on progress and any issues
Key Reporting Metrics:
✓ Total municipalities/councils in level
✓ Files loaded to date
✓ Files remaining
✓ Percentage complete
✓ Expected completion date
✓ Any delays or issues
Procedure Detail: Burndown Chart Interpretation & Report Format▼

Burndown Chart Purpose

Visual chart showing:

  • X-axis: Time period (weeks or dates)
  • Y-axis: Number of files loaded
  • Shows cumulative progress of files loaded
  • Slope indicates loading velocity

Chart Interpretation

  • Steep slope: Files loading quickly, on schedule
  • Flat slope: Few files loaded, potential delays
  • Remaining gap: Files still to be processed

Report Format

End of month report includes:

  • Burndown chart graph (from Register)
  • Summary table: Loaded vs. Remaining count
  • Narrative: Progress update, issues, expected completion
  • Distribution: LGA TL, LRB BM
Input: General Valuation Register with Reval schedule and Burndown chart data
Process: Extract graph and metrics, compile summary text, format report
Decision: Data current? Graph clear? All metrics included?
Output: End of month report to LGA TL and LRB BM with progress snapshot
