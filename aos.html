Q1 – What started this AOS case?

  • Identify trigger source (portal, correspondence, phone, internal action).
  • Confirm customer and land tax account alignment.
  • Record trigger source for audit and correspondence.
  • No eligibility or absentee assessment at this step.
Procedure reference: Viewing the AOS amendment request (p.7–10); Before you begin (p.6)
Input data:
  • Trigger channel: My Land Tax portal / written correspondence / phone call / internally created action item
  • Customer ID
  • Land tax account ID
  • Declaration or amendment details
  • Written items: CM10 document reference
  • Phone items: Phone Calls & Notes log
  • Internal items: Enquiry and Reference tab notes
Processing responsibility:
  • e‑Sys — action item creation and allocation
  • CM10 — document storage
Decision: None — intake only.
Output:
  • AOS action item created
  • Allocated to officer's work tray
  • Trigger source recorded on Enquiry tab

Q2 – Am I allowed to proceed?

  • Check investigation status.
  • Check prior TARB determinations or exemptions.
  • Stop and escalate if restricted.
Do not proceed if eligibility is restricted.
Procedure reference: Before you begin (p.6); Requirements for Applying/Removing the Absentee Owner Surcharge (p.16)
Input data:
  • Active investigation flag (red exclamation mark on customer)
  • Under Investigation non-issue indicator (IC-prefixed investigation number)
  • Prior TARB decisions (objections or private rulings)
  • Phone call history
  • CM10 history
  • High-risk flag if investigated within last 3 years
Processing responsibility:
  • e‑Sys — flag lookup
  • CM10 — investigation file and TARB decision history
Decision: Eligibility gate — Proceed / Halt / Escalate to Investigator.
Output:
  • Active investigation → action item reassigned to investigator
  • Prior TARB denial covers same period → halt and notify customer of previous decision
  • Outside investigation period → proceed with high-risk checks

Q3 – What type of owner is this?

  • Identify Individual, Company, or Trust.
  • For joint owners, classify each owner.
  • Owner type controls applicable absentee rules.
Procedure reference: What is an absentee owner? (p.11); Identifying an absentee owner in e‑Sys (p.14); Joint Ownerships (p.13–14)
Input data:
  • Ownership classification from e‑Sys: Individual / Corporation / Trust
  • Trusts: trust type (Discretionary / Unit / Fixed)
  • Corporations: ASIC company type (Australian / Foreign Co / Non-Company)
  • Joint ownerships: classification of each individual owner
Processing responsibility:
  • e‑Sys — ownership record
  • ASIC — company type verification for corporations
Decision: Owner persona determination — selects applicable rule path:
  • Individual → 4-step test
  • Corporation → incorporation & controlling interest test
  • Trust → beneficiary test
Output:
  • Rule path selected per owner
  • Joint ownerships: each owner classified independently

Q4 – Is there already an absentee owner registration?

  • Check current and historical registrations.
  • Validate effective dates and ownership percent.
  • Read‑only validation step.
Procedure reference: Viewing an Absentee Owner Registration (p.15)
Input data:
  • Current and historical absentee owner registrations in e‑Sys
  • Effective dates
  • Ownership percentage
  • Land taxpayer node presence
Processing responsibility: e‑Sys — registration lookup.
Decision: None — read-only validation.
Output:
  • Registration status: Exists (with dates & percentage) or Not exists
  • Determines whether Q8 action will be a new registration or a modification/cessation

Q5 – Is the owner an absentee owner?

  • Apply owner‑specific absentee tests.
  • Review all declarations and evidence.
  • Escalate if evidence is unclear.
Never assume without evidence.
Procedure reference: Absentee Individual & Four Steps Test (p.11); VISA Clarification (p.12); Absentee Corporation & Controlling Interest (p.12); Absentee Trust — Discretionary, Unit, Fixed (p.13)
Input data:
  • Individuals:
    • Citizenship status (Australian citizen / permanent visa holder / NZ subclass 444 holder)
    • Ordinary residence in Australia
    • Physical presence on 31 December
    • Total absence duration in prior year (6-month threshold)
    • Special rule: NZ 444 holders present in Australia on 26 Feb 2001 = permanent residents
  • Corporations:
    • Country of incorporation (via ASIC — Foreign Co / Non-Company = not Australian)
    • Absentee person controlling interest
  • Trusts:
    • Trust type (Discretionary / Unit / Fixed)
    • Beneficiary list and absentee status of each
    • Discretionary: specified beneficiaries named in trust deed
    • Unit: unitholder disclosure status and PPR beneficiary nomination
Processing responsibility:
  • e‑Sys — absentee flag
  • ASIC — corporation verification
  • SRO Absentee Owner Notification Portal — declarations
Decision: D‑AOS‑01 — Absentee determination per owner type:
  • Individual → 4-step test
  • Corporation → incorporation & controlling interest
  • Trust → beneficiary analysis
Output:
  • Absentee flag set (Yes / No) per owner
  • Trusts: absentee proportion calculated based on beneficiary analysis

Q6 – Does a partial AOS apply?

  • Confirm absentee vs non‑absentee interests.
  • Apply surcharge proportionally.
Procedure reference: Applying a Partial AOS Rate (p.22–23)
Input data:
  • Ownership percentages per owner
  • Absentee flag per owner (from Q5)
  • Trust unitholder proportions (where disclosed)
  • Absentee proportion of interests in land (chain of trusts per s46IAA)
Processing responsibility: e‑Sys — partial rate calculation.
Decision: D‑AOS‑02 — Full surcharge (100% absentee) or partial surcharge (proportional to absentee interest share).
Output:
  • Partial AOS rate applied in e‑Sys reflecting absentee ownership proportion
  • Surcharge rates: 4% from 2024 | 2% for 2020–2023 | 1.5% for 2017–2019 | 0.5% for 2016

Q7 – Is an exemption requested or required?

  • Identify exemption request.
  • Collect supporting documentation.
  • Refer to TARB.
Staff must not approve exemptions.
Procedure reference: Applying an AOS Exemption (p.20–21)
Input data:
  • Exemption application
  • Supporting evidence / documentation
  • Exemption types:
    • Controlling interest in absentee corporation (s3B)
    • Absentee beneficiary of absentee trust (s3BA, from 2018)
  • Gazetted guidelines for exemption decisions
Processing responsibility:
  • Staff — collect documentation and refer
  • TARB — determination authority (staff must not approve or deny)
Decision: Human — TARB determination only.
  • If exemption granted for controlling interest: absentee person taken to not hold that interest → corporation ceases to be absentee
Output:
  • Exemption outcome recorded in e‑Sys (Granted / Denied)
  • If granted: AOS removed or adjusted accordingly

Q8 – How do I apply or remove the AOS?

  • Create or cease registration.
  • Confirm dates and ownership.
  • Trigger recalculation.
Procedure reference: Create the land taxpayer node (p.16); Creating an Absentee Owner Registration (p.17–18); Removing an AOS Indicator (p.19); Recalculating or creating a liability (p.24–25); Check suspense for payments (p.26–27); Close non-issue (p.28); Check assessments (p.29–31)
Input data:
  • Absentee determination results (from Q5)
  • Partial rate (from Q6)
  • Exemption outcome (from Q7)
  • Effective dates (31 December reference date)
  • Land taxpayer node (create if required)
  • Ownership percentage
Processing responsibility:
  • e‑Sys — registration create/cease, liability recalculation
  • Staff — suspense review, payment reallocation, refund initiation
Decision: Execution:
  • Apply AOS → create registration
  • Remove AOS → cease registration
  • Retrospective period: current year plus four preceding years
Output:
  • Updated absentee owner registration in e‑Sys
  • Liabilities recalculated across affected tax years
  • Suspense checked for payments requiring reallocation or refund
  • Non-issue indicators applied/removed as needed
  • Assessments withdrawn, reissued, or newly created as applicable

Q9 – What must be done before closing the action item?

  • Clear suspense.
  • Validate assessments.
  • Issue correspondence.
Procedure reference: Completing the action item (p.32–34); Correspondence (p.35–37)
Input data:
  • Final registration state
  • Recalculated assessments
  • Suspense / refund status
  • Refund ID (if applicable)
  • Transferred customer number and amount (if applicable)
Processing responsibility:
  • Staff — action item completion, root cause selection, comments entry, correspondence
  • e‑Sys — action item status update, checklist, reference tab
  • CM10 — letter storage
  • Communications Manager / Assisted Digital — digital correspondence
  • LetterPro — postal correspondence
Decision: None — procedural closure. Correspondence channel:
  • My Land Tax response → Assisted Digital
  • Contact Us / Smart Form → Assisted Digital
  • Postal letter → only when no digital channel available
Output:
  • Action item status set to Completed
  • Root cause recorded
  • Comments entered (actions taken, result, money transferred/refunded, assessment effect, correspondence method, user ID, date)
  • Refund ID recorded in Reference tab (if applicable)
  • Assessments issued via digital delivery (MLT/BPay View) or extracted for print
  • Correspondence sent to customer confirming AOS outcome
  • Case closed
