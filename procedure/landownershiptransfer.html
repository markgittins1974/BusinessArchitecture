Q1 - Is the customer under active investigation?

  • Check e-Sys Record: Review customer record for investigation indicators and non-issue flags.
  • Identify Investigation Status: Red exclamation mark (!) indicates active investigation or investigation within last 3 years.
  • Verify Investigation Number: Check Under Investigation Non-Issue for IC-prefixed investigation number.
  • Related Customers: Check Joint Ownership members and Trust beneficiaries for investigation flags.
  • Decision: Is customer under investigation or high-risk status?
Do NOT action Land Ownership request if customer is under investigation. Assign to Investigator immediately.
Procedure reference: Before You Begin; Investigation Management
Procedure Detail: Investigation Indicators▼

Active Investigation Flag

Red Exclamation Mark: Visible in e-Sys next to customer number and name. Indicates either active investigation or investigation completed within last 3 years.

High Risk Period

Customers investigated within last 3 years are flagged for 3-year period and should be treated as high risk.

Under Investigation Non-Issue

Will appear on account with investigation number (prefixed by IC, e.g., IC123456). Document this in case notes.

Related Customer Checks

Always check Joint Ownership members and Trust beneficiaries. If related customer is under investigation, do NOT proceed.

Previous Decisions

If property was previously denied by investigation or TARB decision, inform customer of prior decision. Do NOT reprocess.

Source: Before You Begin; Investigation Protocol
Input data: Customer record, investigation status flags, non-issue indicators from e-Sys
Processing responsibility: Assessor (status check), e-Sys (flag visibility), Investigator (case assignment)
Decision: Customer under investigation (YES/NO)
Output: Proceed to Q2 if NO investigation; HALT and reassign to Investigator if YES

Q2 - Can ownership details be verified using authorised sources?

  • Gather Amendment Details: Identify amendment source (customer contact, NOA, internal discovery) and required information.
  • Select Verification Tools: Determine which sources to use: COI, DAS, Landata, Municipal Revaluation Files.
  • Cross-reference Data: Use multiple sources to confirm ownership is accurate and consistent.
  • Document Sources Used: Record which tools verified the ownership and results obtained.
  • Decision: Has ownership been successfully verified?
Do NOT update ownership without verification. If unable to verify, request supporting documents from customer or escalate.
Procedure reference: Confirmation Methods; Verification Process
Procedure Detail: Verification Tools Explained▼

COI - Compliance Operational Intelligence

LTX Property Summary: Enter property number to get quick overview: current e-Sys ownership, municipal data, titles office info, DAS details.

Search Properties: Use folio, SPI, or address for detailed search. Returns registered proprietor/s, registration date, manner of holding, shares, dealing details.

DAS Records Search

Use for conflicting information. Shows transfer documents, transferor/transferee details, settlement dates, transaction types. Important: Registration date ≠ actual transfer date.

Landata Search

Last resort option. Can view NOA images, Transfer of Land instruments, or title history. Note: Some searches incur costs - use after exhausting other options.

Municipal Revaluation Files

Accessed via e-Sys Tools menu. Check council valuation data for property in retrospective period. Verify property references and address match.

Customer Supporting Documents

If verification unsuccessful, request from customer: fully signed contracts, settlement statements, trust deeds, probate, or will copies. Reserve item for 21 days with Non-Issue Indicator.

Source: Confirmation Methods section
Input data: Customer information, amendment source, property references, NOA/documentation
Processing responsibility: Assessor (verification), COI (property data), DAS (transfer records), Landata (title history)
Decision: Ownership verified (YES/NO)
Output: Proceed to Q3 with verified ownership; OR request documentation and place on hold

Q3 - Can ownership records be updated in e-Sys?

  • Load to Tree: Ensure all required customers (transferor and transferee) and properties are loaded to e-Sys tree.
  • Access Transfer Function: Select Ownership node, highlight transferor customer, right-click and select Transfer Property.
  • Enter Transfer Details: Input date of possession, dealing number (if known), and transfer comments (e.g., "Trf'd per DAS").
  • Configure New Owner: Select New Owners tab, associate new customer, set Lead Customer, adjust ownership shares if multiple owners.
  • Apply PPR: If applicable, select PPR exemption option.
  • Complete: Click Complete button to record transfer in e-Sys.
  • Decision: Has ownership been successfully updated?
Ensure ownership shares total 100%. Date of possession: Use DAS settlement date if within retro period, else use SRO standard date (from 2024: 1 Dec 2019).
Procedure reference: Update Ownership of Property in e-Sys
Procedure Detail: Transfer Steps▼

Step 1: Prepare e-Sys Tree

Load all required customers to tree. If property not currently owned by customer, locate via property search and load.

Step 2: Access Transfer Function

Confirm purchasers' current liability. Select Ownership node under property. Highlight transferor. Right-click → Transfer Property.

Step 3: Transfer Details

Date of Possession: Retrospective period = use DAS settlement date. Outside retro = use SRO standard date.

Dealing Number: Enter if known from documentation.

Comments: Record transfer details, e.g., "Trf'd as per DAS. Refer CID 1234567"

Step 4: New Ownership Configuration

Select New Owners tab. Associate Customer (new owner). Choose Lead Customer. If multiple owners, adjust Share% so total = 100%. Select PPR if confirmed.

Step 5: Complete Transfer

Click Complete. Transfer recorded. Proceed to liability recalculation for all affected parties and years.

Source: Update Ownership section
Input data: Verified ownership, transfer date, transferor/transferee details, property record
Processing responsibility: e-Sys (record update), Assessor (data entry), System (validation and storage)
Decision: Ownership update successful (YES/NO)
Output: Updated ownership records; proceed to Q4 liability recalculation

Q4 - Do land tax liabilities require recalculation or creation?

  • Navigate to Liabilities: Select Land Taxpayer node, then select Liabilities.
  • Review Liability Summary: Check current liability status and net liability amounts for all years.
  • Click Recalculate: Click Recalculate button on Liability Summary screen.
  • Select Years: In Recalculate Liability screen, select only years requiring recalculation (NOT years outside retrospective period).
  • Execute Recalculation: Click OK to process the recalculation.
  • Review Outcomes: Check whether liabilities were amended (increase/decrease), withdrawn (removed), or newly created.
  • Check for Penalties: Review Workflow node for any LTX Penalty (Notification Dflt) work items that may have been created.
  • Decision: Have liabilities been appropriately recalculated?
CRITICAL: Do NOT recalculate years outside the retrospective period (5 years: current + 4 prior). Document outcomes for action item notes.
Procedure reference: Recalculating or Creating a Liability
Procedure Detail: Recalculation Outcomes▼

Liability Amendment

Amount increased or decreased for affected year(s). Reflects ownership change or corrected ownership percentage.

Liability Withdrawal

Liability removed entirely from customer. Property no longer in their holdings (e.g., property sold).

New Liability Creation

New liability established for newly identified owner within retrospective period.

Retrospective Period Definition

Current land tax year PLUS four preceding years = 5 years total. Only these years eligible for recalculation.

Penalty Work Items

If recalculation amends liabilities, check Workflow node for LTX Penalty (Notification Dflt) items. Refer to Penalty Tax in e-Sys for Land Tax Procedure for review process.

Joint Ownership Impact

If amendment affects JO customers, recalculate liabilities for all associated members. Changes to one owner impact collective liability.

Source: Recalculating or Creating Liability section
Input data: Updated ownership records, affected years in retrospective period, customer details
Processing responsibility: e-Sys (calculation engine), Assessor (year selection and confirmation), System (recalculation execution)
Decision: Liabilities recalculated successfully (YES/NO)
Output: Updated liability records; penalty work items generated if applicable; proceed to Q5

Q5 - Can assessments be generated or reissued?

  • Check Non-Issue Indicators: Verify no LTX Liability Non-Issue blockers are preventing assessment generation.
  • Review Liabilities: Check Net Liability (outstanding balance) to determine which assessments require generation/reissuance.
  • Select Assessment Years: Identify which years need new or revised assessments from Liabilities node.
  • Generate Assessments: Click Generate Assessment button for each required year.
  • Verify Generation: Confirm assessments have been successfully generated and appear in Assessments node.
  • Note Withdrawn Assessments: Withdrawn assessments will have disappeared from liability list.
  • Decision: Have assessments been appropriately generated or reissued?
Assessment generation blocked by Non-Issue Indicators. Close blockers first: Edit NII, set cease date to day before current date, click OK.
Procedure reference: Generate Assessment/s; Non-Issue Management
Procedure Detail: Assessment Delivery Methods▼

Closing Non-Issue Indicators

If NII blocks assessment generation: Select NII, right-click Edit, enter cease date (day BEFORE today), click OK. Assessment can then be generated next business day.

Reissuing Withdrawn Assessments (Digital Customers)

If assessment withdrawn in error and needs reissue but NOT posted: Generate assessment, select "External - Return To Requestor", ensure Priority NOT selected. Customer only sees last version in My Land Tax.

Reissuing Withdrawn Assessments (Postal Customers)

For customers not registered for digital delivery: Generate assessment WITHOUT "External - Return To Requestor" or "Priority". Email operations@sro.vic.gov.au with extraction request (template: CID, address, Assessment IDs).

Assessment Verification

After generation, verify new/amended assessments appear in Assessments node. Multiple years may require separate generation operations.

Digital vs Postal Delivery

Digital customers (My Land Tax, myGov): Assessments generated automatically notify customer. Postal customers: Assessment printed and mailed via operations team.

Source: Generate Assessment section; Assessment Delivery
Input data: Recalculated liabilities, NII status, years requiring assessment, customer delivery method
Processing responsibility: e-Sys (assessment generation), Assessor (year selection), System (validation and delivery)
Decision: Assessments generated/reissued successfully (YES/NO)
Output: Generated/reissued assessments; proceed to correspondence and action item completion

Procedure Summary

  • Identifies and manages land ownership amendments in e-Sys (adding or removing land from customer holdings)
  • Establishes nine-step decision process from investigation check through assessment generation
  • Requires verification using COI, DAS, Landata, and municipal records before updating ownership
  • Defines liability recalculation and assessment generation procedures for affected years
  • Covers complex scenarios including Joint Ownership, Trusts, and Company Share Units
  • Concludes with customer correspondence and action item completion

Land Ownership Amendment Sources

Customer-Initiated Amendments

Created when customer contacts via phone, My Land Tax, or written request:

  • Property not listed in their assessment
  • Property has been sold (no longer owned)
  • Ownership percentage is incorrect
  • Property was never owned by them

Internally-Created Amendments

Identified by SRO staff or automated processes during related tasks:

  • Valuations team discovery
  • Section 105 certificate processing
  • Debt Management Services referral
  • Investigation findings
  • Late NOA processing post-assessment

Special Transfers

  • Terms Sale: 15% deposit paid
  • Life Tenancy: Based on Will provisions
Processing Note: Basic property withdrawals processed via AD/MLT. Complex amendments (trusts, estates, shares) require manual e-Sys processing.

Verification Method Hierarchy

Use verification tools in sequence:

Priority Tool Information
1 COI Property Summary Current ownership, municipal data, titles office info, DAS details
2 COI Search Properties Proprietor/s, registration date, manner of holding, shares, dealings
3 DAS Records Transfer documents, transferor/transferee details, settlement dates
4 Municipal Revaluation Council valuation data, property references, address history
5 Landata NOA images, TOL instruments, title history (costs apply)
Customer Documentation: If verification unsuccessful, request supporting documents and place item on 21-day hold with Non-Issue Indicator.

Complex Scenarios

Company Share Units

Individuals hold shares in company owning land/buildings. Company on title; unit holders are deemed owners for land tax.

Verification: Search ASIC for company. Check Share Details (Historic view). Match ownership dates with other sources. Request supporting docs if needed.

Note: ASIC for verification only. Process transfer normally after confirming ownership.

NICO Plan (Not in Common Ownership)

Subdivision/consolidation of multiple titles with different owners. After NICO, all properties held jointly.

Treatment: Assess as JO in e-Sys. Apportion shares by parent title areas. Remains JO until new titles lodged.

Example: Property 1: 85,780 sqm (Owner A); Property 2: 10,750 sqm (Owner B). Totals 96,530 sqm. A: 88.864%; B: 11.136%

Definitions & Key Acronyms

AD/MLT - Assisted Digital/My Land Tax • COI - Compliance Operational Intelligence • DAS - Document Assessing System • JO - Joint Ownership • LTX - Land Tax • NII - Non-Issue Indicator • NOA - Notice of Acquisition • PPR - Principal Place of Residence • SRO - State Revenue Office • TARB - Technical Advice and Review Branch • TOL - Transfer of Land

Retrospective Period: Current year plus four prior years (5 years total) • Land Ownership Amendment: Adding/removing land from customer's holdings • Transfer: Moving property ownership from one customer to another • Investigation: Compliance review of customer's land tax obligations
