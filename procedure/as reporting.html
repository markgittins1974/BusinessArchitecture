Land Tax AOS Exemption Reports - Procedure Overview

Purpose

Compile and present AOS exemption data to meet the Treasurer's statutory obligation to report to Parliament on exemptions granted under the Land Tax Act.

Background

  • AOS (Absentee Owner Surcharge): 0.5% surcharge (2016), 1.5% (2017-2019), 2.0% (2020+) on absentee-owned land
  • Exemption Authority: Treasurer can exempt corporations (s3B(2)), trusts can be exempt (s3BA(2)) from 2018
  • Reporting Requirement: Treasurer must report exemptions to Parliament every 6 months (minimum)
  • TARB Role: Technical Advice and Review Branch handles all exemption applications and reporting

Two Annual Reports

  1. Half-Year Report (March): Period 1 July - 31 December (tabled in Parliament)
  2. Full Year Report (September): Full financial year (tabled in Parliament)
Authority: Division: Policy Advisory and Legal Services | Branch: Technical Advice and Review | Revised: 5 February 2025
Input: e-Sys MC988 reports (exemption data), Objections Summary Report
Process: Q1-Q4: Extract → Calculate → Prepare → Submit & Publish
Decision: Data correct? Exemptions unique? Rolled-over accounted? Parliament tabling deadline met?
Output: Ministerial brief, Parliamentary letters, exemptions schedule (tabled and published)

Q1 - Extract Exemption Data from e-Sys

MC988 Report Generation

  1. Navigate to: H:\TechnicalAdviceReview\Review\COMMON\Esys\Reports\[month]
  2. Locate MC988 report (auto-generated end of each month)
  3. Report extracts ALL AOS exemptions with following data:
    • Customer ID and name
    • TARB decision date
    • TARB file reference
    • Exemption start and end dates
    • Land tax group indicator
    • Taxable land value (current + prior 4 years)
    • Tax foregone (current + prior 4 years)
    • Indicator if no taxable landholdings

Objections Summary Report

  1. Generate from e-Sys 'ML-240' screen
  2. Use criteria:
    • Rev Line: Land Tax
    • Grounds: Ltx – Absentee Owner Surcharge – Treasurer's discretion
    • Date Resolved: [as required]
  3. Report shows all closed TARB files
  4. Cross-check against MC988 to ensure all decisions actioned in e-Sys
CRITICAL: If TARB decisions NOT yet actioned by Operations, escalate to Land Revenue Branch Team Leader immediately.

MC988 Report Path

H:\TechnicalAdviceReview\Review\COMMON\Esys\Reports\[month]

Data Elements in MC988

  • Customer ID (unique identifier)
  • Customer name (full name as recorded)
  • TARB decision date (date exemption approved)
  • TARB file reference (unique file ID)
  • Exemption start date (when exemption begins)
  • Exemption end date (if exemption expires)
  • Land tax group indicator (Y/N if in group)
  • Land tax group customer ID (if applicable)
  • Taxable land value (current year and prior 4 years)
  • Land tax foregone (current year and prior 4 years)
  • Indicator if no taxable landholdings

Data Availability Note

Tax foregone and land values only populated for years where exemption granted. Prior years show zero if exemption didn't apply in those years.

Input: e-Sys MC988 report (auto-generated monthly), Objections Summary Report
Process: Extract exemption records, verify all TARB decisions actioned in Operations
Decision: All records extracted? All decisions actioned? Objections report matches?
Output: Clean dataset ready for Q2 processing

Q2 - Calculate Exemption Counts and Values

Establishing Number of Exemptions Granted

Key Concept: One exemption may affect multiple customers (land tax groups). Each year counts as a new exemption (rolled-over annually unless material change notified).

FOR HALF-YEAR REPORT (1 July - 31 Dec):

  1. From MC988 report, identify all exemptions with 'TARB decision date' within the period
  2. Remove duplicates: Keep only one per 'TARB file reference' (prefer removing zero tax foregone)
  3. Count total number of years with tax foregone > 0
  4. Cross-check against Objections Summary Report (ensure all TARB files actioned on e-Sys)
  5. If gaps found, escalate to Land Revenue Branch Team Leader

FOR FULL YEAR REPORT (Full Financial Year):

PLUS account for "rolled-over" exemptions (granted prior year, still active current year):

  1. Identify all exemptions with 'TARB decision date' BEFORE the period
  2. Remove duplicates: Keep only one per 'TARB file reference'
  3. Count exemptions where current year tax foregone > 0
  4. Add to new exemptions count from steps above
  5. Cross-check against Objections Summary Report

Establishing Value of Exemptions Granted

Value = total tax foregone (land tax waived due to exemption).

FOR HALF-YEAR REPORT:

  1. From MC988 report, identify all exemptions with 'TARB decision date' within period
  2. Add up total tax foregone for current year and prior 4 years (for those exemptions)

FOR FULL YEAR REPORT:

PLUS add rolled-over exemption value:

  1. Identify exemptions with 'TARB decision date' BEFORE the period
  2. Add up total tax foregone for CURRENT YEAR ONLY (for those exemptions)
  3. Add to new exemptions value from steps above
Important: Preference for removing duplicates should be given to exemptions with zero tax foregone.
Input: MC988 dataset, Objections Summary Report
Process: Count new exemptions, identify rolled-over, calculate total tax foregone value
Decision: All duplicates removed? All rolled-over accounted? Values calculated correctly?
Output: Number of exemptions and total value for reporting period

Q3 - Prepare Parliamentary Documentation

Four Documents Required

1. Ministerial Brief to Treasurer

  • Signature by: Branch Manager, Executive Director, Deputy Commissioner
  • Template available from: PLB (Parliamentary Liaison Branch)
  • Contains: Exemption count, total value, reporting period

2. Letter to Clerk of Legislative Council

  • Check current details on Parliamentary website
  • Transmits: Exemptions schedule for tabling

3. Letter to Clerk of Legislative Assembly

  • Check current details on Parliamentary website
  • Transmits: Exemptions schedule for tabling

4. Two Copies of Exemptions Schedule

  • Lists all exemptions granted in reporting period
  • CRITICAL: Schedule must NOT identify any particular customer
  • Contains only: Exemption count, total value, no customer names or details
CRITICAL: None of the documentation should identify any particular customer by name, address, or ID.
Procedure Detail: Exemptions Schedule Format▼

Exemptions Schedule Contents

The schedule tabled in Parliament contains aggregate information only:

  • Period covered (e.g., 1 July 2024 - 31 December 2024)
  • Number of exemptions granted
  • Total value of exemptions (land tax foregone)
  • No individual customer details, names, or addresses
  • No property information
  • Aggregate data only

Format Example

Schedule for period 1 July - 31 December 2024:
Number of exemptions granted: [X]
Total value of land tax foregone: $[Amount]

Input: Exemption count, total value from Q2
Process: Prepare ministerial brief, Parliamentary letters, exemptions schedule
Decision: All documents complete? No customer identifiers? Treasurer signature ready?
Output: Complete documentation package ready for submission

Q4 - Submit to Parliament and Publish

Timing Requirements

Critical Deadlines:

  • Half-Year Report (1 Jul - 31 Dec): Forward to Treasurer by mid-March
  • Full Year Report (Full FY): Forward to Treasurer by mid-September
  • Documentation must reach Treasurer with sufficient time for Parliament tabling

Parliament Tabling Process

  1. Ministerial brief submitted to Treasurer for signature (Branch Manager, Executive Director, Deputy Commissioner)
  2. Treasurer tables copies of exemptions schedule in both Houses of Parliament
  3. Schedule becomes part of Parliamentary record

Publishing on SRO Website

  1. Once exemptions schedule tabled in Parliament, advise Corporate Affairs
  2. Corporate Affairs publishes schedules on SRO website
  3. Information becomes publicly available
Timeline Note: Ensure documentation forwarded to Treasurer early enough for tabling within statutory timeframe. Mid-March for half-year, mid-September for full year.
Input: Complete documentation package (brief, letters, schedules)
Process: Forward to Treasurer, tabled in Parliament, published on website
Decision: Deadline met? Corporate Affairs notified? Website updated?
Output: Report tabled in Parliament, information published, statutory obligation met

Definitions & Acronyms

Treasurer's Discretion - Authority to exempt from AOS under Land Tax Act
